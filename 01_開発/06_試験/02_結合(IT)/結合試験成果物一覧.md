# 🧩 結合試験工程 成果物一覧

以下に、**システム開発における結合試験工程（Integration Test）**で、業務チームおよび開発チーム双方が作成・確認する主な**成果物一覧**を整理します。
（※ウォーターフォール型を基本とした一般的な構成です）

---

## 🧩 結合試験工程 成果物一覧

### ① 計画系成果物

| 成果物名                               | 概要                                       | 作成主体           | 確認主体     |
| ---------------------------------- | ---------------------------------------- | -------------- | -------- |
| **結合試験計画書（Integration Test Plan）** | 結合試験の目的、対象範囲、環境構成、実施体制、スケジュール、合否基準などを定義。 | 開発チーム（テストリーダー） | 業務チーム／PM |
| **結合試験項目一覧（Test Item List）**       | 結合対象モジュール、試験観点、優先度を一覧化。                  | 開発チーム          | 業務チーム／QA |
| **結合試験環境構成図**                      | 試験に使用するサーバ、ネットワーク、データベース、外部IFなどの構成。      | インフラ担当         | 開発・業務チーム |

---

### ② 設計系成果物

| 成果物名                                        | 概要                          | 作成主体  | 確認主体     |
| ------------------------------------------- | --------------------------- | ----- | -------- |
| **結合試験仕様書（Integration Test Specification）** | 各テスト項目ごとの入力データ、手順、期待結果を定義。  | 開発チーム | 業務チーム／QA |
| **テストデータ設計書**                               | 結合試験に使用するデータの設計・作成方針。       | 開発チーム | 業務チーム    |
| **結合観点マトリクス（結合パターン表）**                      | モジュール間の結合点・インターフェースの組合せを整理。 | 開発チーム | 業務チーム／QA |

---

### ③ 実施系成果物

| 成果物名                                | 概要                                | 作成主体     | 確認主体     |
| ----------------------------------- | --------------------------------- | -------- | -------- |
| **結合試験実施記録（Test Execution Report）** | 各試験項目の実施結果・判定・証跡（スクリーンショット、ログなど）。 | 開発チーム    | 業務チーム／QA |
| **不具合管理表（バグトラッキングリスト）**             | 試験中に発見された不具合の記録、原因、修正状況。          | 開発チーム／QA | 業務チーム    |
| **修正依頼票／修正版リリースノート**                | 不具合修正内容と修正版提供情報。                  | 開発チーム    | 業務チーム    |

---

### ④ 評価・完了系成果物

| 成果物名                                           | 概要                                | 作成主体     | 確認主体     |
| ---------------------------------------------- | --------------------------------- | -------- | -------- |
| **結合試験結果報告書（Integration Test Summary Report）** | 試験全体の実施状況、完了率、不具合傾向、判定結果、今後の対応方針。 | 開発チーム／QA | 業務チーム／PM |
| **試験完了確認書**                                    | 結合試験完了の正式合意書（次工程遷移の根拠）。           | 開発チーム    | 業務チーム／PM |
| **品質評価報告書（必要に応じて）**                            | 不具合密度・再現率・品質評価などの定量的分析。           | QAチーム    | PM／経営層   |

---

### ⑤ 補助資料（任意／推奨）

| 成果物名                     | 概要                             |
| ------------------------ | ------------------------------ |
| **結合試験スケジュール表（ガントチャート）** | 試験項目ごとの実施期間、担当者、依存関係を明確化。      |
| **ログ／トレース取得手順書**         | エラー発生時の調査のためのログ取得方法を明記。        |
| **環境リカバリ手順書**            | テスト環境破損時に迅速に復旧するための手順。         |
| **インターフェース一覧表**          | 各モジュール間のI/F項目、型、制約、プロトコルなどを整理。 |

---

## 🔄 関連工程との対応関係

| 前工程（単体試験）        | 対象工程（結合試験）           | 後工程（総合試験）         |
| ---------------- | -------------------- | ----------------- |
| モジュール単体の動作確認     | モジュール間の連携・インターフェース検証 | システム全体の業務シナリオ検証   |
| 成果物：単体試験仕様書・実施記録 | 成果物：結合試験仕様書・結果報告書    | 成果物：総合試験仕様書・結果報告書 |
