# 🧾 受入試験工程 成果物一覧
以下は、**システム開発における「受入試験（UAT: User Acceptance Test）」工程で業務チームが関与・作成する成果物一覧**です。
ウォーターフォール開発を前提に、一般的なプロジェクト管理・品質管理の観点で体系的に整理しています。

---

## 🧾 受入試験工程の成果物一覧

### ① 試験計画関連

| 成果物名                | 内容概要                                   | 作成者           | 承認者    |
| ------------------- | -------------------------------------- | ------------- | ------ |
| **受入試験計画書（UAT計画書）** | 試験の目的、対象範囲、体制、スケジュール、実施方針、環境、判定基準などを定義 | 業務チーム（主導）、PMO | 発注者責任者 |
| **試験体制図**           | 試験実施者・業務担当者・開発側サポートなどの役割分担表            | PM/業務リーダー     | PMO    |

---

### ② 試験設計関連

| 成果物名               | 内容概要                             | 作成者   | 承認者         |
| ------------------ | -------------------------------- | ----- | ----------- |
| **受入試験仕様書（試験項目表）** | 要件定義書や業務シナリオをもとに、試験項目・手順・期待結果を整理 | 業務チーム | 発注者責任者      |
| **試験データ設計書**       | 試験に使用するデータ（マスタ・トランザクション）一覧と投入方法  | 業務チーム | 開発チーム（レビュー） |
| **試験環境設定手順書**      | 環境構築や初期データ投入の手順を整理               | 開発チーム | 業務チーム       |

---

### ③ 試験実施関連

| 成果物名                     | 内容概要                          | 作成者                 | 承認者        |
| ------------------------ | ----------------------------- | ------------------- | ---------- |
| **試験実施記録（試験結果報告書）**      | 実施日、担当者、結果（合否）、不具合No.、備考などを記録 | 業務担当者               | 試験責任者      |
| **試験証跡（スクリーンショット・出力結果）** | 試験実施のエビデンス。画面キャプチャ・帳票・ログなど    | 業務担当者               | PMO（必要に応じ） |
| **不具合管理票（障害票）**          | 試験中に発生した不具合を記録・分類・追跡          | 業務チーム（登録）・開発チーム（対応） | PM         |

---

### ④ 試験結果整理・評価関連

| 成果物名                 | 内容概要                       | 作成者      | 承認者    |
| -------------------- | -------------------------- | -------- | ------ |
| **不具合分析報告書**         | 不具合件数、傾向、重要度別の集計・分析        | PMOまたはQA | 発注者    |
| **受入判定報告書（試験完了報告書）** | 試験結果、合否判定、残課題、受入可否をまとめた報告書 | 業務チーム責任者 | 発注者責任者 |
| **残課題一覧（移行・運用対応項目）** | 本番前に持ち越す課題やリリース後対応項目の整理    | 業務チーム    | PMO    |

---

### ⑤ 関連資料（補助成果物）

| 成果物名                   | 内容概要                     | 作成者       |
| ---------------------- | ------------------------ | --------- |
| **業務シナリオ一覧／テストケース対応表** | 要件・シナリオ・テスト項目の対応関係を整理    | 業務チーム     |
| **受入試験進捗管理表**          | 試験項目ごとの進捗・合否を管理する一覧表     | 業務リーダー    |
| **試験環境利用計画表**          | 試験環境・帳票・データセットの利用スケジュール  | PMO       |
| **試験終了報告書（総括版）**       | 試験全体の所感、改善点、次回プロジェクトへの提言 | PMO／業務責任者 |

---

## 🧩 成果物の体系構造（例）

```
受入試験
 ├─ ①試験計画
 │    └─ 受入試験計画書、体制図
 ├─ ②試験設計
 │    ├─ 試験仕様書
 │    ├─ 試験データ設計書
 │    └─ 環境設定手順書
 ├─ ③試験実施
 │    ├─ 試験実施記録
 │    ├─ 試験証跡
 │    └─ 不具合管理票
 ├─ ④結果整理・評価
 │    ├─ 不具合分析報告書
 │    └─ 受入判定報告書
 └─ ⑤補助資料
      ├─ シナリオ対応表
      ├─ 進捗管理表
      └─ 環境利用計画表
```
