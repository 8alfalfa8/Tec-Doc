# 📘 単体テスト工程 成果物一覧

以下は、**システム開発工程における「単体テスト工程」で作成・使用される成果物一覧**です。
業務チーム／開発チーム双方で確認・管理する想定で整理しています。

---

## 📘 単体テスト工程 成果物一覧

| 区分            | 成果物名                          | 概要                                  | 作成主体        | 備考                          |
| ------------- | ----------------------------- | ----------------------------------- | ----------- | --------------------------- |
| **1. 計画系**    | 単体テスト計画書（UT計画書）               | テストの目的、対象範囲、実施方針、担当、スケジュールなどを定義     | 開発チーム       | 基本設計・詳細設計を基に策定              |
| **2. 設計系**    | 単体テスト仕様書（UT仕様書）               | 各プログラム／モジュール単位でのテストケース、条件、期待値を定義    | 開発チーム       | 関数・メソッドレベルで詳細に設計            |
|               | テストデータ設計書                     | テストケースに使用する入力データと期待出力を整理            | 開発チーム       | ExcelやCSV、SQLなどで定義される場合あり   |
|               | モック・スタブ設計書                    | 外部I/Fや依存モジュールを模擬するモジュールの設計書         | 開発チーム       | 外部依存の多いモジュールで使用             |
| **3. 実行系**    | 単体テスト実施記録（実施結果報告書）            | 各テストケースの実施日、担当、結果（OK/NG）、不具合内容を記録   | 開発チーム       | テストツールやスプレッドシートで管理される場合も多い  |
|               | 単体テストエビデンス（ログ・スクリーンショット・出力結果） | テスト実行時の結果証跡（コンソール出力、画面キャプチャ、DB結果など） | 開発チーム       | テスト自動化の場合は自動生成されることも        |
| **4. 不具合管理系** | 不具合報告書（バグ票）                   | テスト中に発見された不具合を報告・管理                 | 開発チーム       | Redmine, JIRA, Backlogなどで管理 |
|               | 不具合修正履歴                       | 修正内容、リビジョン、再テスト結果を記録                | 開発チーム       | ソース管理システムと連動                |
| **5. 評価・完了系** | 単体テスト結果報告書                    | テスト完了後にまとめる最終報告。テスト件数、合格率、残課題などを記載  | 開発チーム       | プロジェクト管理者へ提出                |
|               | レビュー記録（UT設計レビュー・結果レビュー）       | テスト仕様書や結果のレビュー記録                    | 開発チーム／QAチーム | 品質保証プロセスの一環                 |

---

## 💡 補足：自動化対応の場合の追加成果物

| 成果物名                                    | 概要                        |
| --------------------------------------- | ------------------------- |
| テストスクリプト（JUnit, pytest, etc.）           | テスト仕様書に基づく自動化コード          |
| テスト実行レポート（HTML, XML等）                   | 自動テスト実行後に出力されるサマリレポート     |
| CI設定ファイル（GitHub Actions, Jenkinsfile 等） | 自動テストを継続的に実行するためのパイプライン定義 |
