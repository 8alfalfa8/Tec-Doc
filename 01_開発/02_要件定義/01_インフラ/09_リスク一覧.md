# 📘 リスク一覧（実務例）

以下に、**金融機関向け システム基盤プロジェクト用 リスク一覧（実務例）** を提示します。<br>
準拠観点：FISC安全対策基準、金融庁監督指針

※要件定義〜設計フェーズで利用できる管理台帳形式です。

---
# 1. リスク管理方針

* 影響度 × 発生可能性で評価
* 高リスクは経営報告対象
* 月次レビュー実施
* 残余リスク明確化

---

# 2. リスク評価基準

| 区分 | 影響度      | 発生可能性 |
| -- | -------- | ----- |
| 高  | 全停止／信用失墜 | 高確率   |
| 中  | 一部停止     | 可能性あり |
| 低  | 軽微影響     | 低確率   |

# 7. 実務で必須の観点

✔ 「DR未検証」は必ず明文化<br>
✔ 「権限過剰」は必ず入れる<br>
✔ 「設定誤り」はIaCで統制<br>
✔ 「容量逼迫」は監視＋予測<br>
✔ 「外部依存リスク」は明示

---

# 8. レビュー観点（要件定義時）

* 単一障害点は残っていないか
* RTO/RPOは経営承認済みか
* ログ保存期間は監査合意済みか
* セキュリティ棚卸頻度は具体的か
* DR試験は計画に入っているか

# 2. リスク評価基準

## 2.1 影響度定義

| レベル | 定義               |
| --- | ---------------- |
| 高   | 全停止、重大な信用毀損、法令違反 |
| 中   | 一部機能停止、業務遅延      |
| 低   | 軽微な影響            |

## 2.2 発生可能性定義

| レベル | 定義      |
| --- | ------- |
| 高   | 発生確率が高い |
| 中   | 発生可能性あり |
| 低   | 発生確率が低い |

## 2.3 リスク評価方法

* リスク値 ＝ 影響度 × 発生可能性
* 高リスクは経営報告対象
* 月次レビュー実施
