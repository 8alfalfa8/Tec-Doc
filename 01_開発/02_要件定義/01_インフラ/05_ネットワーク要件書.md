# 2. ネットワーク構成方針

## 2.1 基本原則

1. 多層防御構成（Defense in Depth）
2. 業務系／情報系完全分離
3. 本番／検証／開発環境の物理的または論理的完全分離
4. 全通信ホワイトリスト制御
5. 内部通信含め暗号化

---

## 2.2 論理構成（概要）

構成階層：

* 外部接続層（Internet／閉域網）
* DMZ層
* Web層
* AP層
* DB層
* 運用管理層（踏み台／監視）

---

# 3. 環境分離要件

| 区分      | 分離方式    | 備考      |
| ------- | ------- | ------- |
| 本番－開発   | VPC完全分離 | ルート共有禁止 |
| 本番－DR   | 別リージョン  | 物理的分離   |
| 業務系－情報系 | FW分離    | 相互通信禁止  |

クラウド基盤：Amazon Web Services

* 本番：東京リージョン
* DR：大阪リージョン

---

# 4. IPアドレス設計

## 4.1 アドレス体系

| セグメント | CIDR          | 用途   |
| ----- | ------------- | ---- |
| DMZ   | 10.10.0.0/24  | 外部公開 |
| Web   | 10.10.1.0/24  | Web層 |
| AP    | 10.10.2.0/24  | AP層  |
| DB    | 10.10.3.0/24  | DB層  |
| 管理    | 10.10.10.0/24 | 運用   |

* RFC1918準拠
* 将来拡張を見据え/22確保

---

# 5. 通信制御要件

## 5.1 制御原則

* 初期設定：Deny All
* 必要通信のみ許可
* ポート固定

---

## 5.2 通信一覧（抜粋例）

| No | 送信元      | 宛先  | Port | Protocol | 暗号化     |
| -- | -------- | --- | ---- | -------- | ------- |
| 1  | Internet | Web | 443  | HTTPS    | TLS1.2+ |
| 2  | Web      | AP  | 8443 | HTTPS    | TLS1.2+ |
| 3  | AP       | DB  | 1521 | TCP      | 内部TLS   |

* SSH直接接続禁止
* 管理アクセスは踏み台経由のみ

---

# 6. 冗長化要件

## 6.1 回線

* キャリア二重化
* 別経路引込

## 6.2 機器

* FW：Active-Standby
* L3SW：VRRP構成
* ロードバランサ：冗長

## 6.3 切替時間

* 通信断発生後5分以内復旧

---

# 7. 外部接続要件

## 7.1 インターネット接続

* WAF設置
* IDS/IPS導入
* DDoS対策

## 7.2 閉域網接続

* 勘定系センター接続：専用線
* IPsec暗号化
* 相互証明書認証

---

# 8. セキュリティ統制

## 8.1 FWポリシー管理

* 変更時承認必須
* 変更履歴保存7年以上
* 四半期棚卸し実施

## 8.2 管理アクセス

* 踏み台サーバ経由
* 多要素認証
* 操作ログ取得

---

# 9. 監視・ログ要件

## 9.1 監視対象

* 回線死活
* 帯域使用率70%以上アラート
* 異常トラフィック検知

## 9.2 ログ保存

| 種別        | 保存期間 | 保存先     |
| --------- | ---- | ------- |
| FWログ      | 7年以上 | 改ざん防止領域 |
| Flow Logs | 3年   | 集中ログ基盤  |

---

# 10. 時刻同期

* 全NW機器NTP同期
* 冗長NTPサーバ構成
* 時刻誤差±1秒以内

---

# 11. DR関連ネットワーク要件

* 本番⇔DR専用回線確保
* 切替手順書整備
* 年1回実切替試験

---

# 12. 性能要件（NW観点）

| 項目        | 要件         |
| --------- | ---------- |
| 最大同時セッション | 200,000    |
| ピーク帯域     | 5Gbps      |
| レイテンシ     | 10ms以内（内部） |

---

# 13. リスク管理（例）

| No | リスク       | 影響   | 対策      |
| -- | --------- | ---- | ------- |
| 1  | 単一回線障害    | 全停止  | 二重化     |
| 2  | FW誤設定     | 通信断  | 承認フロー強化 |
| 3  | 想定外トラフィック | 性能劣化 | 帯域拡張余力  |

* ✔ DR疎通試験を年次実施
* ✔ 通信一覧は設計凍結前に確定

# 1. 目的・適用範囲

本書は、対象システムのネットワーク構成・分離方針・通信制御・冗長化・監査要件を定義し、安全かつ可用性の高い通信基盤を確立することを目的とする。
