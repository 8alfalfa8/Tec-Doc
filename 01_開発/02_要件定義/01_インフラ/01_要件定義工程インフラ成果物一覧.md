# 🧭 インフラ要件定義成果物一覧

以下に、**システム開発の要件定義工程において「インフラ領域」で作成・整理される主な成果物一覧**を示します。
これは、ウォーターフォール開発を基本としつつ、クラウド（AWS/Azure/GCPなど）やオンプレ構成を含む一般的な構成を想定したものです。

---

## 🧭 1. インフラ要件定義の目的

* システム要件（機能・非機能）を踏まえ、**必要なインフラ環境（構成・性能・セキュリティ・運用）を明確化**する。
* 以降の設計・構築・テストの基礎となる。

---

## 📘 2. 成果物一覧（カテゴリ別）

### 【A. 全体構成・方針系】

| 成果物名           | 内容                                        | 作成目的                |
| -------------- | ----------------------------------------- | ------------------- |
| インフラ要件定義書      | システム全体構成、要件、前提条件、制約事項を整理                  | 開発・運用・セキュリティの共通認識形成 |
| インフラ構成方針書      | クラウド/オンプレ選定、環境分離方針、命名規則、IP設計方針など          | 設計以降の指針明確化          |
| システム構成図（論理構成図） | システム全体の構成（Web/AP/DB/ネットワーク/外部連携）を論理レベルで図示 | 関係者間の認識統一           |

---

### 【B. 環境定義・構成系】

| 成果物名                         | 内容                                   | 作成目的             |
| ---------------------------- | ------------------------------------ | ---------------- |
| 環境一覧表                        | 開発・検証・本番などの各環境の役割、構成、利用目的            | 環境ごとの差異を整理       |
| サーバ一覧表                       | 各サーバの用途、スペック（CPU/メモリ/OS/ミドルウェア）      | サイジングや見積の基礎資料    |
| クラウドサービス利用一覧（AWS/Azure/GCP等） | 利用予定のマネージドサービス一覧（RDS, S3, ECS, etc.） | コスト・セキュリティ要件検討   |
| ネットワーク構成図（論理/物理）             | サブネット/VPC/ファイアウォール/ルーティング構成          | 通信要件・セキュリティ設計の基礎 |
| IPアドレス設計書                    | 各機器・環境のIP割当計画                        | ネットワーク構築計画の基礎    |

---

### 【C. 性能・容量・可用(信頼)性系】

| 成果物名           | 内容                                       | 作成目的         |
| -------------- | ---------------------------------------- | ------------ |
| サイジング要件書       | 処理量、トランザクション数、アクセス頻度を踏まえたCPU/メモリ/ストレージ算定 | 適切なリソース見積    |
| 性能要件定義書        | 応答時間、スループット、同時接続数、バッチ処理時間など              | SLA・性能試験基準設定 |
| 可用(信頼)性・冗長化要件書     | 冗長構成、フェイルオーバー要件、RTO/RPO                  | 障害対策・運用基準明確化 |
| バックアップ・リストア要件書 | 取得周期、保存期間、対象データ、リストア手順レベル                | データ保全設計      |

---

### 【D. セキュリティ・運用系】

| 成果物名        | 内容                           | 作成目的              |
| ----------- | ---------------------------- | ----------------- |
| セキュリティ要件定義書 | 認証・認可・暗号化・ログ管理・アクセス制御        | 設計・監査の前提定義        |
| アクセス制御設計方針書 | 管理者/利用者権限、VPN・IAM構成、ゼロトラスト方針 | 権限管理の統一           |
| 運用要件定義書     | 監視・ジョブ管理・障害対応・運用分担           | 運用設計の前提           |
| 監視要件一覧      | 監視対象（CPU、メモリ、ログ、アプリ死活など）と閾値  | 運用監視設計の基礎         |
| ログ要件一覧      | 収集対象ログ、保存期間、監査要件             | セキュリティ・トレーサビリティ確保 |

---

### 【E. 外部要件・非機能系】

| 成果物名       | 内容                      | 作成目的         |
| ---------- | ----------------------- | ------------ |
| 非機能要件定義書   | 性能・可用性・拡張性・保守性などの非機能全般  | インフラ設計の前提明確化 |
| 外部連携要件一覧   | 他システムとの通信方式・ポート・プロトコル   | 接続・テストの準備    |
| 認証・認可方式設計書 | SSO、LDAP、Keycloak、SAML等 | 統合認証設計の前提資料  |

---

### 【F. コスト・契約系】

| 成果物名              | 内容                  | 作成目的          |
| ----------------- | ------------------- | ------------- |
| インフラ見積書（初期/ランニング） | 機器・サービス利用料・運用費の試算   | 予算計画・経営判断資料   |
| ライセンス管理一覧         | OS・DB・MWのライセンス種別と数量 | コンプライアンス・費用管理 |

---

### 【G. IaC・構成管理準備系（クラウド・DevOps対応）】

| 成果物名            | 内容                           | 作成目的       |
| --------------- | ---------------------------- | ---------- |
| IaC適用範囲定義書      | Terraform/Ansible等で自動化する範囲定義 | 構築自動化方針明確化 |
| CI/CDパイプライン構成概要 | デプロイ・検証・リリースの自動化設計概要         | DevOps基盤整備 |
| Gitリポジトリ構成方針    | IaCコード・設定管理方針                | バージョン管理統一  |

---

## 📄 3. 成果物一覧（まとめ）

| 区分        | 主な成果物                           |
| --------- | ------------------------------- |
| 全体構成・方針   | インフラ要件定義書、構成方針書、システム構成図         |
| 環境構成      | 環境一覧表、サーバ一覧表、クラウド利用一覧、ネットワーク構成図 |
| 性能・可用(信頼)性    | サイジング要件、性能要件、可用(信頼)性・冗長化要件、バックアップ要件 |
| セキュリティ・運用 | セキュリティ要件、アクセス制御、監視・ログ要件、運用要件    |
| 外部・非機能    | 非機能要件定義書、外部連携要件、認証設計書           |
| コスト・契約    | 見積書、ライセンス管理一覧                   |
| IaC・構成管理  | IaC範囲定義書、CI/CD構成概要、リポジトリ方針      |

※なお、オンプレミスシステムの場合、データベース要件定義書も必要となります。
