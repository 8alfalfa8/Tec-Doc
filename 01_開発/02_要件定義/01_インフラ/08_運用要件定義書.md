# 📘 運用要件定義書（実務例）

以下に、**金融機関向け 運用要件定義書（実務例）** を提示します。<br>
想定：インターネットバンキング基盤<br>
準拠：FISC安全対策基準、金融庁監督指針<br>
基盤例：Amazon Web Services

※実務提出レベルの具体度で記載しています。

---

# 1. 文書概要

| 項目     | 内容           |
| ------ | ------------ |
| 文書名    | 運用要件定義書      |
| 対象システム | インターネットバンキング |
| 対象環境   | 本番／DR／検証／開発  |
| 作成部門   | IT運用部        |
| 承認     | CIO／IT統括部長   |

---

# 2. 目的

本書は、対象システムの安定稼働を実現するために必要な運用体制・運用プロセス・監視・保守要件を定義することを目的とする。

---

# 3. 運用体制

## 3.1 体制図

* 運用統括責任者
* 運用リーダー
* 一次対応（24/365）
* 二次対応（基盤担当／アプリ担当）
* セキュリティ統括（CISO配下）

## 3.2 役割分担

| 役割      | 責任範囲      |
| ------- | --------- |
| 運用部     | 監視・障害一次対応 |
| 基盤部     | インフラ障害対応  |
| 開発部     | アプリ改修     |
| セキュリティ部 | インシデント対応  |

---

# 4. 監視要件

## 4.1 監視対象

* サーバ（CPU／メモリ／ディスク）
* DB
* ネットワーク
* ミドルウェア
* アプリケーション
* セキュリティログ
* バックアップ成否

## 4.2 アラート基準

| 項目    | 閾値   | 通知先 |
| ----- | ---- | --- |
| CPU   | 80%  | 運用  |
| Disk  | 75%  | 運用  |
| DB接続数 | 80%  | 基盤  |
| エラー率  | 1%以上 | アプリ |

* 重大アラート：30分以内初動対応
* 監視：24時間365日

---

# 5. 障害管理

## 5.1 障害区分

| 区分 | 定義   | 目標復旧時間 |
| -- | ---- | ------ |
| S1 | 全停止  | 1時間以内  |
| S2 | 一部停止 | 4時間以内  |
| S3 | 軽微   | 翌営業日   |

## 5.2 対応フロー

1. 検知
2. 切り分け
3. 一次対応
4. 恒久対策立案
5. 再発防止レビュー

---

# 6. 変更管理

## 6.1 変更区分

| 区分 | 例         | 承認    |
| -- | --------- | ----- |
| 緊急 | セキュリティパッチ | 部長承認  |
| 通常 | 構成変更      | CAB承認 |
| 定例 | 月次メンテ     | 事前通知  |

* 本番直接変更禁止
* IaC管理必須

---

# 7. パッチ管理

| 区分 | 適用期限   |
| -- | ------ |
| 重大 | 72時間以内 |
| 高  | 2週間以内  |
| 中  | 月次適用   |

* 事前検証必須
* 適用後確認手順明確化

---

# 8. バックアップ・リストア運用

| 項目       | 要件  |
| -------- | --- |
| 日次バックアップ | 実施  |
| 成否確認     | 毎日  |
| 復元試験     | 年1回 |
| 別拠点保管    | 実施  |

---

# 9. ログ管理

| 種別       | 保存期間 |
| -------- | ---- |
| 取引ログ     | 7年以上 |
| セキュリティログ | 7年以上 |
| OSログ     | 3年以上 |

* 改ざん防止措置
* 時刻同期必須

---

# 10. 性能管理

* 月次レポート提出
* リソース使用率分析
* 将来容量予測
* ボトルネック特定

---

# 11. セキュリティ運用

* 権限棚卸（四半期）
* 脆弱性診断（四半期）
* 年1回外部診断
* 標的型攻撃訓練実施

---

# 12. DR運用

* 年1回切替訓練
* 手順書整備
* 切替時間測定
* 課題管理

---

# 13. 定例報告

| 項目       | 頻度  |
| -------- | --- |
| 稼働報告     | 月次  |
| 障害報告     | 随時  |
| セキュリティ報告 | 四半期 |
| 容量報告     | 半期  |

---

# 14. 委託先管理

* SLA明文化
* 定例レビュー（月次）
* セキュリティ監査（年次）
* NDA締結

---

# 15. 教育・訓練

* 年1回セキュリティ教育
* 運用手順書更新教育
* 障害訓練（年1回）

---

# 16. リスク管理

| リスク    | 対策    |
| ------ | ----- |
| 属人化    | 手順書整備 |
| 監視漏れ   | 二重監視  |
| パッチ未適用 | 管理台帳  |
| DR未検証  | 年次試験  |

---

# 🔎 実務上の重要ポイント

* ✔ 運用体制は「名前」まで決める
* ✔ 障害区分と復旧時間は経営承認
* ✔ DR訓練は実地で実施
* ✔ 変更管理は監査対象
* ✔ 監視閾値は暫定値で終わらせない
