# 📘 セキュリティ要件書（実務例）

以下に、**金融機関向け セキュリティ要件書（実務レベル記載例）** を提示します。<br>
想定：インターネットバンキング更改プロジェクト<br>
準拠：FISC安全対策基準、金融庁監督指針、PCI Security Standards Council（該当時）

---

# 1. 文書概要

| 項目   | 内容              |
| ---- | --------------- |
| 文書名  | セキュリティ要件書       |
| 対象   | インターネットバンキング基盤  |
| 対象環境 | 本番／DR／検証／開発     |
| 作成部門 | IT基盤部・セキュリティ統括部 |
| 承認   | CISO／IT統括部長     |

---

# 2. セキュリティ基本方針

1. 多層防御（Defense in Depth）
2. ゼロトラスト原則適用
3. 最小権限の原則
4. 全通信暗号化
5. 全操作ログ取得・保全
6. 設計段階からのセキュリティ組込み（Shift Left）

---

# 3. セキュリティ管理体制

| 区分    | 内容          |
| ----- | ----------- |
| 統括責任者 | CISO        |
| 運用責任者 | セキュリティ運用責任者 |
| 監査    | 内部監査部／外部監査  |

* 年1回以上の内部監査実施
* 重大インシデントは経営報告

---

# 4. 認証・認可要件

## 4.1 利用者認証

* 多要素認証必須
* ワンタイムパスワード採用
* パスワードポリシー：

  * 12文字以上
  * 英大小文字・数字・記号混在
  * 90日有効期限

## 4.2 管理者認証

* 個人ID付与（共用禁止）
* 多要素認証必須
* IP制限＋踏み台経由

---

# 5. アクセス制御要件

* RBAC方式採用
* 職務分掌徹底（開発・運用分離）
* 権限棚卸し：四半期ごと

---

# 6. 通信セキュリティ要件

| 項目   | 要件           |
| ---- | ------------ |
| 外部通信 | TLS1.2以上     |
| 内部通信 | 原則TLS暗号化     |
| VPN  | IPsec AES256 |
| 証明書  | 公開鍵2048bit以上 |

* 脆弱プロトコル（SSL、TLS1.0/1.1）禁止

---

# 7. ネットワークセキュリティ

* 多層FW構成
* WAF導入
* IDS/IPS導入
* DDoS対策必須
* 東西トラフィック監視

クラウド利用時：Amazon Web Services

* Security Group最小化
* NACL明示設定
* VPC Flow Logs有効

---

# 8. サーバ・OSセキュリティ

## 8.1 ハードニング

* CISベンチマーク準拠
* 不要サービス停止
* root直接ログイン禁止
* SSH鍵認証必須

## 8.2 パッチ管理

| 区分 | 適用期限   |
| -- | ------ |
| 重大 | 72時間以内 |
| 高  | 2週間以内  |
| 中  | 月次     |

---

# 9. マルウェア対策

* EDR導入
* リアルタイム検知
* 定義ファイル自動更新
* USB制御（原則禁止）

---

# 10. ログ管理要件

## 10.1 取得対象

* OS操作ログ
* DBアクセスログ
* 管理者操作ログ
* FWログ
* アプリ操作ログ

## 10.2 保存要件

| 種別       | 保存期間 |
| -------- | ---- |
| 取引ログ     | 7年以上 |
| セキュリティログ | 7年以上 |
| 通常操作ログ   | 3年以上 |

* 改ざん防止（WORM等）
* 時刻同期必須（NTP）

---

# 11. 脆弱性管理

* 年1回以上の第三者診断
* 四半期ごとの内部診断
* 重大脆弱性即時是正
* Webアプリ診断必須

---

# 12. インシデント対応

## 12.1 CSIRT体制

* 24時間以内初動対応
* 重大時は経営報告
* 原因分析・再発防止策策定

## 12.2 対応フロー

1. 検知
2. 影響範囲特定
3. 封じ込め
4. 復旧
5. 再発防止

---

# 13. データ保護要件

## 13.1 暗号化

| 対象     | 方式     |
| ------ | ------ |
| 保存データ  | AES256 |
| バックアップ | 暗号化必須  |
| 鍵管理    | HSM利用  |

## 13.2 個人情報保護

* マスキング実装
* 本番データの開発利用禁止

---

# 14. クラウド利用時追加要件

* IAM最小権限
* APIキー定期ローテーション
* CloudTrail有効化
* ルートアカウント未使用

---

# 15. DRセキュリティ

* DR環境も本番同等セキュリティ
* バックアップ暗号化
* 年1回復旧訓練

---

# 16. 教育・訓練

* 年1回以上のセキュリティ教育
* 標的型攻撃メール訓練実施
* 管理者向け専門研修

---

# 17. リスク管理

| リスク        | 対策     |
| ---------- | ------ |
| 権限過剰付与     | 定期棚卸   |
| 設定誤り       | 二重承認   |
| 内部不正       | 操作ログ監視 |
| サプライチェーン攻撃 | 委託先評価  |

---

# 🔎 実務上の重要ポイント

* ✔ 設計段階で通信一覧と権限一覧を固定
* ✔ ログ保存期間は必ず監査部と合意
* ✔ DR環境を忘れない
* ✔ 権限棚卸は形式的にしない
* ✔ 本番データ持ち出し禁止を明文化
