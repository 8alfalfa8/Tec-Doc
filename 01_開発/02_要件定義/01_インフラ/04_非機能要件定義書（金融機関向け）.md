# 📘 非機能要件定義書（実務例）

以下に、**金融機関向け 非機能要件定義書（実務記載例サンプル）** を提示します。<br>

準拠：FISC安全対策基準、金融庁監督指針

※数値はあくまで実務例です。

---

# 1. 文書概要

| 項目   | 内容             |
| ---- | -------------- |
| 文書名  | 非機能要件定義書       |
| 対象   | インターネットバンキング基盤 |
| 対象環境 | 本番／DR／検証／開発    |
| 版数   | v1.0           |
| 承認   | IT統括部長／監査部長    |

---

# 2. 可用性要件（実記載例）

## 2.1 SLA

| 項目     | 要件値      | 備考     |
| ------ | -------- | ------ |
| 月間稼働率  | 99.99%以上 | 計画停止除外 |
| 年間停止時間 | 52分以内    |        |

算出式：
（総稼働時間－障害停止時間）÷総稼働時間

---

## 2.2 RTO / RPO

| 項目  | 要件    |
| --- | ----- |
| RTO | 2時間以内 |
| RPO | 15分以内 |

* DBは同期レプリケーション採用
* DRサイトは別地域配置
* 年1回実切替訓練実施

---

## 2.3 冗長化

* Web/AP層：マルチAZ構成
* DB層：Active-Standby同期構成
* FW：Active-Standby冗長
* 回線：キャリア二重化

---

# 3. 性能要件（実記載例）

## 3.1 トランザクション性能

| 指標      | 要件           |
| ------- | ------------ |
| 最大同時接続数 | 15,000       |
| ピークTPS  | 1,200 TPS    |
| 平均応答時間  | 2秒以内（95%タイル） |
| 最大応答時間  | 5秒以内         |

想定ピーク：給与振込日 9:00〜11:00

---

## 3.2 バッチ処理

* 日次締処理完了時刻：翌日3:00まで
* リラン機能必須
* 障害時再開可能設計

---

## 3.3 負荷試験基準

* 想定ピーク1.3倍負荷で2時間連続実施
* CPU使用率80%未満
* メモリ枯渇なし

---

# 4. 拡張性要件

* 年率利用者増加率：10%
* 水平スケール可能設計
* リソース追加時間：30分以内
* DB容量上限：初期10TB → 将来30TB対応

---

# 5. セキュリティ要件

## 5.1 認証

* 管理者：多要素認証必須
* 一般利用者：ワンタイムパスワード
* 本番アクセス：事前承認制

---

## 5.2 暗号化

* 通信：TLS1.2以上
* 保存：AES256
* 鍵管理：HSM利用

---

## 5.3 脆弱性管理

* 月次脆弱性診断
* 重大脆弱性は72時間以内対応
* 年1回第三者診断実施

---

# 6. ログ・監査要件

## 6.1 取得対象

* 管理者操作ログ
* DBアクセスログ
* OS操作ログ
* FW通信ログ
* アプリ監査ログ

---

## 6.2 保存期間

| 種別   | 保存期間 |
| ---- | ---- |
| 操作ログ | 7年以上 |
| 監査ログ | 10年  |
| 通信ログ | 3年   |

* 監査部門専用参照権限付与
