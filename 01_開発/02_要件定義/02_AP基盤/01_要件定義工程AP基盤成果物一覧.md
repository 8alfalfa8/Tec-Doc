## 🧩 AP基盤における要件定義工程の成果物一覧


システム開発の**要件定義工程**において、**AP基盤（アプリケーション基盤）**として整理・作成される成果物の一覧を以下に体系的にまとめます。

---

## 1. **全体構成・方針関連**

| 成果物名                    | 内容                                    | 目的                     |
| ----------------------- | ------------------------------------- | ---------------------- |
| AP基盤要件定義書（またはAP基盤要件定義書） | システム全体の非機能要件・共通機能・運用要件を整理             | システム全体に共通するアプリ基盤要件の明確化 |
| アプリケーションアーキテクチャ構成図      | アプリ層・ミドル層・外部連携構成の概要図                  | 構成の可視化と関係者間の共通理解形成     |
| 技術選定方針書（または技術要件整理書）     | 使用言語、FW、ライブラリ、開発ツール、クラウドサービス等の選定根拠を記載 | 技術的統一性・維持管理性の確保        |

---

## 2. **機能横断（共通基盤）要件**

| 成果物名               | 内容                                   | 目的                |
| ------------------ | ------------------------------------ | ----------------- |
| 認証・認可要件定義書         | 認証方式（例：OAuth2、SAML）、権限モデル、ID管理方針など   | セキュリティとユーザ管理の共通化  |
| ログ管理要件定義書          | アプリログ、操作ログ、監査ログの出力内容、出力方式、保存期間       | 監査対応・トレーサビリティ確保   |
| 例外処理／エラー管理要件定義書    | 例外ハンドリング、メッセージ出力方針、コード体系             | 障害発生時の一貫した対応方針    |
| ジョブ管理要件定義書         | バッチ処理、スケジューラ連携方式、ジョブ管理方式             | 安定稼働と運用性向上        |
| 通信・API共通要件定義書      | REST/SOAP仕様、タイムアウト、リトライ制御、共通Header定義 | サービス間連携の統一        |
| ファイル入出力要件定義書       | ファイル形式、命名規則、文字コード、転送方式               | 外部連携・バッチ連携の標準化    |
| キャッシュ・セッション管理要件定義書 | セッション維持方式、キャッシュ層構成、TTL設定方針           | パフォーマンスとセキュリティ最適化 |
| メッセージ管理要件定義書       | メッセージID、表示文言、国際化（i18n）対応方針           | 多言語化対応・共通メッセージ管理  |

---

## 3. **非機能要件対応**

| 成果物名        | 内容                            | 目的              |
| ----------- | ----------------------------- | --------------- |
| 性能・容量要件定義書  | 同時接続数、レスポンスタイム、スループット、データ量見積  | インフラ・ミドル設計の基礎資料 |
| 可用性要件定義書    | 冗長構成、フェイルオーバー要件、RTO/RPO定義     | システム継続性の確保      |
| 保守運用要件定義書   | ログローテーション、監視項目、アラート閾値、運用ツール連携 | 運用効率の最大化        |
| セキュリティ要件定義書 | 認証・通信暗号化・入力チェック・脆弱性対策など       | 安全なアプリ運用の確立     |

---

| テスト環境構成要件定義書   | テスト環境構成、データ準備方式、テスト自動化方針                  | 検証環境の標準化     |

| 共通部品設計方針書      | ロガー、DBアクセス層、ユーティリティなどの共通部品方針              | 再利用性・保守性の向上  |

## 🧭 参考：要件定義工程におけるAP基盤の目的

* システム全体で共通化できる仕組み（認証、ログ、例外処理など）の明確化
* 開発チーム間で技術統一を図り、品質・運用性を確保
* 非機能要件（性能・可用性・セキュリティ）の設計基礎を確立
* 次工程（基本設計・詳細設計）への橋渡しとなるアーキテクチャを確定
