# 🧭 要件定義工程成果物一覧

以下に **システム開発における要件定義工程（要件定義フェーズ）** の一般的な **成果物一覧** を、目的別・分類別に整理して示します。
（ウォーターフォール型を基本とし、アジャイル型にも応用できる構成です）

---

## 🧭 要件定義工程の目的

* システムが実現すべき **業務要件・機能要件・非機能要件** を明確化し、
  以降の設計・開発・テストの基礎となる仕様を確定させる。

---

## 📘 成果物一覧（カテゴリ別）

### ① 全体計画・方針関連

| 成果物名      | 内容概要                | 作成目的         |
| --------- | ------------------- | ------------ |
| プロジェクト計画書 | 開発体制・進行計画・リスク管理方針   | 開発全体の方向性共有   |
| スコープ定義書   | 開発対象範囲・除外範囲・前提条件    | プロジェクト境界の明確化 |
| 要件定義方針書   | 要件定義の進め方、レビュー・承認ルール | 要件定義作業の統一化   |

---

### ② 業務要件関連

| 成果物名                  | 内容概要              | 作成目的         |
| --------------------- | ----------------- | ------------ |
| 現行業務調査報告書（As-Is）      | 現行システム・業務プロセスの分析  | 改善対象の理解      |
| 業務課題整理表               | 現行業務上の問題・要改善点の一覧  | 改善要件の抽出      |
| 業務フロー図（As-Is / To-Be） | 業務手順・情報フローの可視化    | 新業務モデルの設計    |
| 業務要件定義書               | 業務プロセス・担当者・入出力の定義 | システム化対象業務の確定 |

---

### ③ 機能要件関連

| 成果物名               | 内容概要             | 作成目的        |
| ------------------ | ---------------- | ----------- |
| 機能一覧表（機能構成図）       | システムで提供する機能の全体構成 | 機能範囲の明確化    |
| 機能要件定義書            | 各機能の入力・出力・処理内容   | 実装仕様の基本定義   |
| 画面一覧／画面遷移図         | 画面構成と画面間の関係      | UI/UXの整理    |
| 画面レイアウト案（ワイヤーフレーム） | 主要画面の構成・項目配置     | 画面要件の確認     |
| 帳票一覧・帳票レイアウト案      | 出力帳票の種類・項目       | 帳票機能の定義     |
| 外部インタフェース仕様書       | 他システムとの連携仕様      | システム間整合性の確保 |

---

### ④ 非機能要件関連

| 成果物名       | 内容概要                    | 作成目的          |
| ---------- | ----------------------- | ------------- |
| 非機能要件定義書   | 性能、信頼性、運用性、保守性、セキュリティなど | システム品質基準の定義   |
| 性能要件定義     | レスポンス時間、スループットなど        | 性能検証の基準明確化    |
| セキュリティ要件定義 | 認証、認可、暗号化、監査ログなど        | 情報保護対策の明示     |
| 可用性・冗長化要件  | 稼働率、障害復旧時間など            | サービス継続性確保     |
| 運用・保守要件定義  | バックアップ、監視、運用手順など        | 運用フェーズへの引継ぎ準備 |

---

### ⑤ データ・インタフェース関連

| 成果物名                  | 内容概要                     | 作成目的       |
| --------------------- | ------------------------ | ---------- |
| データ定義書（論理データモデル）      | エンティティ、属性、リレーション         | データ構造の明確化  |
| データ項目定義書              | 各項目の型・長さ・制約条件            | データ品質の担保   |
| 外部インタフェース一覧           | 接続対象システム、方向、方式           | 連携仕様の統一化   |
| API仕様書（REST/GraphQL等） | APIエンドポイント、リクエスト/レスポンス仕様 | 連携開発の共通仕様化 |

---

### ⑥ トレーサビリティ・整合性管理関連

| 成果物名            | 内容概要                | 作成目的      |
| --------------- | ------------------- | --------- |
| 要件トレーサビリティマトリクス | 要件 ⇔ 機能 ⇔ テスト項目の対応表 | 漏れ・重複の防止  |

| 要件変更履歴表         | 要件変更の内容と理由          | 変更管理・影響把握 |

---

### ⑦ 承認・管理関連

| 成果物名        | 内容概要             | 作成目的      |
| ----------- | ---------------- | --------- |
| 要件定義レビュー記録書 | レビュー内容・指摘事項・対応状況 | 品質保証の記録   |
| 要件定義承認書     | 顧客・ステークホルダーによる承認 | 要件凍結の証跡   |
