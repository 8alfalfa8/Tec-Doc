# 🧩 運用保守工程 成果物一覧
以下は、**システム開発における「運用保守」工程**で一般的に作成・維持される成果物の一覧を体系的に整理したものです。
（対象：業務システム・Webシステム等の運用保守フェーズ）

---

## 🧩 1. 運用設計関連成果物

| 分類             | 成果物名                           | 主な内容           | 作成/更新タイミング |
| -------------- | ------------------------------ | -------------- | ---------- |
| 運用設計書          | 運用体制・手順・監視・障害対応など全般を定義         | 運用開始前に策定、定期見直し |            |
| 運用手順書（運用マニュアル） | 定常作業（バックアップ、ジョブ実行、ログ確認など）の手順   | 初期運用前、手順変更時    |            |
| 監視設計書          | 監視項目、閾値、通知方法、運用監視ツール設定         | 運用前            |            |
| ジョブ設計書／バッチ管理表  | バッチ処理やジョブスケジュールの定義             | 運用前、変更時        |            |
| バックアップ・リストア設計書 | バックアップ周期、世代管理、リストア手順           | 運用前、環境変更時      |            |
| キャパシティプラン      | CPU/メモリ/ディスク/ネットワークの利用見通しと閾値管理 | 運用前・定期更新       |            |
| 運用カレンダー        | 定常業務、メンテナンス、停止期間などのスケジュール      | 年次・月次更新        |            |

---

## ⚙️ 2. 保守設計・変更管理関連成果物

| 分類           | 成果物名                       | 主な内容       | 作成/更新タイミング |
| ------------ | -------------------------- | ---------- | ---------- |
| 保守計画書        | 保守体制、SLA、対応区分（軽微改修・障害対応など） | 運用開始前      |            |

| 変更管理台帳       | プログラム修正、設定変更、リリース履歴        | 各変更ごとに更新   |            |
| バージョン管理表     | ソース・設定・DBスキーマ等のバージョン履歴     | 継続更新       |            |
| リリース計画書      | リリース手順、影響範囲、実施スケジュール       | 各リリース時     |            |
| テスト証跡（保守テスト） | 修正箇所のテスト結果・検証ログ            | 改修ごとに作成    |            |

### ✅ まとめ分類（運用保守成果物マトリクス）

| 区分       | 主目的          | 主な成果物例          |
| -------- | ------------ | --------------- |
| 運用設計     | 安定稼働を支える基盤整備 | 運用設計書、監視設計書、手順書 |
| 保守管理     | 改修・障害対応の統制   | 変更管理台帳、保守計画書    |
| 障害管理     | 障害の早期検知と再発防止 | 障害記録票、分析報告書     |
| セキュリティ管理 | 不正防止・監査対応    | 権限管理表、ログ管理設計書   |
| 構成管理     | 環境変更・構成追跡    | 構成図、設定一覧        |
| 運用報告     | SLA・品質管理     | 月次報告、SLAレポート    |
| 教育・引継ぎ   | 運用継続性確保      | 教育資料、引継ぎ資料      |
