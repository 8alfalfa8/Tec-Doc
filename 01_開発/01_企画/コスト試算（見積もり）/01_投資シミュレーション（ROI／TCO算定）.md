# ◆投資シミュレーション（ROI／TCO算定）の詳細説明

以下は「投資の意思決定に使える実務レベル」の手順、入力情報、作業タスク、計算方法、成果物、および検証・提示のポイントを一通りまとめたものです。最後に**簡単な数値例（計算手順を含む）**も付けますので、モデルの作り方と結果の見せ方が具体的に分かります。

---

## 1) 目的とスコープ決定（前提）

* **目的**：何を評価したいか（コスト削減効果の検証、新システム導入の採算可否、クラウド移行のTCO比較など）。
* **期間**：通常は3〜7年（長期投資なら10年など）。評価期間を明確に。
* **対象範囲**：どの費用／効果を含めるか（ハード／ソフト／人件費／運用費／リプレース費等）。
* **ディスカウント率（割引率）**：WACC、社内要求利回り、または保守的に5〜10%などを設定。
* **通貨・物価前提**：円建て・インフレ前提（インフレ率、税金影響の扱い）。

---

## 2) 必要な入力情報（作成条件・データ）

### キャッシュアウト（コスト）

* 初期投資（設備費、ライセンス、導入ベンダー費用、移行費用）
* 年次運用コスト（保守、SaaS費、クラウド課金、人件費）
* 更新・追加投資（ハード刷新、ライセンス更新）
* 廃棄・撤去コスト（必要なら）

### キャッシュイン（効果／便益）

* 直接的コスト削減（削減された人件費、ライセンス削減など）
* 収益増加（新機能による売上増）
* 運用効率化の価値（作業時間短縮 × 時間単価）
* リスク低減価値（ダウンタイム削減の期待値、罰則回避）
* 税制優遇や補助金（あれば）

### その他

* 残存価値（評価期間終了時の資産価値）
* 税金・減価償却の扱い（後述）
* 感度分析で使うパラメータ（効果の±％、ディスカウント率のレンジなど）

---

## 3) 作成タスク（ワークフロー）

1. **要件ヒアリング**：経営・事業・IT運用から前提値を収集。
2. **データ収集**：既存の財務データ、労務データ、クラウド/オンプレの利用ログ等を取得。
3. **コスト項目の洗い出し**：表形式で年次ごとに整理（CapEx / OpEx）。
4. **便益の定量化**：時間削減→金額換算、売上向上の根拠作成。
5. **キャッシュフローモデル作成**：年次CF表（投資年度を t=0 とする）。
6. **財務計算**：NPV、IRR、簡易ROI、回収期間、TCO算定。
7. **感度分析／シナリオ分析**：ベース・楽観・悲観シナリオ、パラメータ別感度。
8. **リスク評価**：想定外コスト、導入遅延、技術リスクの定性的評価。
9. **成果物作成**：レポート、要点スライド、Excelモデル（入力セルと出力セルを分離）。
10. **レビュー＆プレゼン**：経営層向けの意思決定資料作成。

---

## 4) 作成方法・計算手法（実務で使う主要指標）

* **TCO（Total Cost of Ownership）**
  期間中に発生するすべてのコストの合計（CapEx + OpEx + 廃棄費用など）を年次で積算する。割引して現在価値で表すことが多い（Discounted TCO）。

* **NPV（正味現在価値）**
  全ての将来キャッシュフロー（便益 − コスト）を割引率で現在価値に換算して合算。NPV > 0 → 投資を検討可。
  
  <img width="112" height="41" alt="image" src="https://github.com/user-attachments/assets/938aad21-e2cb-4104-9123-d8913a1deec2" />


  [
  \mathrm{NPV} = \sum_{t=0}^{N} \frac{CF_t}{(1+r)^t}
  ]

  （ただし (CF_0) は通常マイナスの初期投資）

* **IRR（内部収益率）**
  NPV = 0 となる割引率。IRR が社内要求利回りを上回れば採算性が高い。

* **単純ROI（累積ベース）**

  [
  \mathrm{ROI} = \frac{\text{期間中の総便益} - \text{期間中の総コスト}}{\text{期間中の総コスト}}
  ]

  ※割引考慮していないため補助的指標。

* **回収期間（Payback）**
  キャッシュインが投資を回収するまでの年数（割引前／割引後）。

* **感度分析**
  主要パラメータ（効果額、人件費単価、割引率、導入遅延日数）を上下に変えてNPV/IRRの変動を確認。

* **シナリオ分析**
  複数シナリオ（楽観／ベース／悲観）を作成し、意思決定のロバスト性を示す。

---

## 6) モデル設計上の実務ポイント・落とし穴

* **透明性**：前提セル（入力）は明確に。前提を変更すれば何が変わるか分かるように。
* **タイミング**：効果発現のタイミングを正確に（導入初年度は未整備で効果が低い等）。
* **税・減価償却の扱い**：税引き後CFで精緻に評価する場合は税効果を反映。
* **人的影響の過小評価**：定量化が難しい効果（UX改善、従業員満足）は注記で示す。
* **維持費の抜け漏れ**：運用監視、バックアップ費、人件費を忘れがち。
* **SaaSの単価変化**：ユーザー数増加での課金増をシミュレーション。
* **端数・四捨五入の扱い**：提示用は丸めるがモデルは小数保持。

## 7) Excelでの実装（簡易テンプレート）

* **Sheet: Inputs**

  * 割引率 r
  * 期間 N
  * 初期投資（CapEx）
  * 年次運用コスト（年1〜N）
  * 年次便益（年1〜N）
  * 残存価値（年N）
* **Sheet: Cashflow**

  * 各年の CF = 便益 − コスト（t=0 は -初期投資）
  * 割引係数 = (1+r)^t
  * 割引後CF = CF / (1+r)^t
  * NPV = SUM(割引後CF)
* **Sheet: Analysis**

  * IRR 関数、Payback（累積CF）
  * Sensitivity table（縦: 割引率、横: 効果変化）
* **Sheet: Charts**

  * NPVバー、累積CFグラフ、TCO比較折れ線

## 8) 実務で有用なチェックリスト

* 前提は誰が合意したか（事業責任者／財務）
* データの出所を明記（例：給与単価＝総務データ）
* 感度分析は最低「±10%」「±20%」で実施
* 最重要パラメータを上位3つに絞って検証
* 最終レポートに「意思決定に必要な追加情報」を明記

## 9) 【具体例】簡単な数値モデルと計算（手順を示す）

**前提**

* 初期投資（t=0） = **1,200,000 円**
* 年間の便益（コスト削減） = **400,000 円 / 年**（年1〜5）
* 評価期間 = **5 年**（t=1..5）
* 割引率 = **5.0%（0.05）**

**目的**：NPV・単純ROI・回収期間・IRRを算出する。

### (A) 各年の割引後便益（年ごとの現在価値）

割引係数は ( (1 + r)^t )。具体的に計算します（小数第以下は四捨五入して円にしています）。

* Year 1（t=1）:
  割引係数 = 1.05
  現在価値 = (400,000 \div 1.05 = 380,952.3809...) → **380,952 円**

* Year 2（t=2）:
  割引係数 = (1.05^2 = 1.1025)
  現在価値 = (400,000 \div 1.1025 = 362,811.7914...) → **362,812 円**

* Year 3（t=3）:
  割引係数 = (1.05^3 = 1.157625)
  現在価値 = (400,000 \div 1.157625 = 345,535.0394...) → **345,535 円**

* Year 4（t=4）:
  割引係数 = (1.05^4 = 1.21550625)
  現在価値 = (400,000 \div 1.21550625 = 329,080.9899...) → **329,081 円**

* Year 5（t=5）:
  割引係数 = (1.05^5 = 1.2762815625)
  現在価値 = (400,000 \div 1.2762815625 = 313,410.4666...) → **313,410 円**

**割引後便益合計（現在価値）** = 380,952 + 362,812 + 345,535 + 329,081 + 313,410 = **1,731,790 円**

### (B) NPV（正味現在価値）

NPV = 割引後便益合計 − 初期投資
= 1,731,790 − 1,200,000 = **531,790 円**

→ **NPV > 0** なので、割引後の観点では投資は採算に合う（この前提下）。

### (C) 単純ROI（期間累積、割引考慮なし）

* 期間中の総便益（割引なし） = 400,000 × 5 = **2,000,000 円**
* 期間中の総コスト（単純） = 初期投資 = **1,200,000 円**（ここでは運用コストを0と仮定）
* 単純ROI = (2,000,000 − 1,200,000) / 1,200,000 = 800,000 / 1,200,000 = **0.666666... = 66.67%（5年合計）**

### (D) 回収期間（割引前）

累積便益：
年1: 400,000（累積 400,000）
年2: 800,000（累積 800,000） → 1,200,000 を超えるのは年3に到達（年3の途中）
実際の回収期間（線形補間） = 2 + (400,000 / 400,000) = 3.0 年（この単純例だとちょうど年3で回収）

### (E) IRR（内部収益率：NPV=0となる割引率）

* このキャッシュフロー（-1,200,000, +400,000 ×5）に対して数値的に求めると IRR ≒ **19.86% / 年**（内部収益率）。
  （解法は数値法［ニュートン法や二分探索］で求めます。IRR が社内期待利回りより高ければ採算性良し）

---

## 10) 提示方法（経営への説明）

* 結論（採る／採らない）を最初に示し、要点（NPV、IRR、回収期間）を並べる。
* 前提が変われば結果がどう動くか（感度分析の主要3パターン）を必ず添付。
* リスクと不確定要素（導入遅延・効果不達成）を数値化できる範囲で添える。
* Excelモデルを別添で渡し、入力セルだけ変更すれば再試算できるようにする。

---

## まとめ（実務的な提言）

* 初回は**ベースシナリオ＋楽観・悲観の3シナリオ**を作って経営に示す。
* モデルの透明性（何を入れて何を除外したか）を最重視する。
* 感度分析を用いて「どのパラメータが最も結果に効くか」を示し、投資判断に必要な追加データやガバナンス事項を提示する。
