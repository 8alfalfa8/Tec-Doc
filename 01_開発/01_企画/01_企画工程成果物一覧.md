# 🧭 システム開発における「企画工程」成果物一覧

以下に「**システム開発における企画工程（上流工程の最上流）**」で一般的に作成される**成果物一覧**を体系的に整理します。
対象はウォーターフォール開発・アジャイル開発いずれにも共通する基本構成を踏まえています。

---

## 🧭 システム開発における「企画工程」とは

**目的**：

* システム化の必要性と目的を明確化
* 対象業務や課題の整理
* 投資判断（ROI）や費用対効果の検討
* プロジェクト実行の可否判断（Go / No-Go）

---

## 📘 企画工程の成果物一覧

| 区分                | 成果物名                    | 主な内容                                              | 作成目的・利用場面            |
| ----------------- | ----------------------- | ------------------------------------------------- | -------------------- |
| **1. 背景・目的定義**    | ① システム化構想書（または構想企画書）    | - システム導入の背景<br>- 現状課題と改善方針<br>- システム化の目的・範囲       | 経営層やステークホルダーへの企画説明資料 |
|                   | ② 現状分析報告書               | - 現行業務プロセス（As-Is）<br>- 課題・問題点整理<br>- 既存システム構成     | 問題の明確化・再発防止策検討の基礎資料  |
| **2. 業務分析・課題整理**  | ③ 業務フロー図（As-Is / To-Be） | 現行業務および理想業務の流れを図式化                                | 業務の非効率・改善余地を可視化      |
|                   | ④ 業務課題一覧表               | 各業務ステップにおける課題・要因・影響度・優先度                          | 課題解決方針の整理            |
| **3. システム化構想・方針** | ⑤ システム化方針書              | - 対象範囲（業務・機能）<br>- システム構成（概略アーキテクチャ）<br>- 投資方針・期間 | 経営判断用の企画概要資料         |
|                   | ⑥ システム概要図               | サービス・業務・データの流れを俯瞰図で表現                             | 関係者間の共通理解形成          |
| **4. 投資・効果検討**    | ⑦ 費用対効果分析書（ROI分析）       | - 投資コスト試算<br>- 効果試算（人件費削減・品質向上・リスク低減等）            | 経営判断（承認会議）用資料        |
|                   | ⑧ 概算見積書                 | ハード／ソフト／開発／保守費用の概算                                | 予算策定および投資判断資料        |
| **5. 実施体制・計画**    | ⑨ プロジェクト体制図             | プロジェクト組織、役割、責任分担                                  | 開発体制を明確化             |
|                   | ⑩ プロジェクト基本計画書           | - スケジュール（マイルストーン）<br>- リスク・課題管理方針<br>- 品質・変更管理方針  | 企画承認後の着手計画           |
| **6. 要求整理（初期段階）** | ⑪ システム要求一覧（ハイレベル要求）     | - 機能要求・非機能要求の概要<br>- 優先度付け                        | 要件定義へのインプット資料        |
|                   | ⑫ 利用シナリオ概要（ユースケース概要）    | ユーザ視点での利用イメージ                                     | 要件定義フェーズで詳細化する前段階    |
| **7. 意思決定・承認**    | ⑬ 企画承認申請書（稟議書）          | 投資額、目的、効果、リスク、実施体制を明記                             | 経営層・意思決定会議で承認を得る     |
| **8. 補助資料**       | ⑭ 関係者一覧／ステークホルダーマップ     | 利害関係者と影響度を整理                                      | 合意形成および調整計画に利用       |
|                   | ⑮ リスク・課題一覧              | 想定リスクと対応策                                         | 企画段階でのリスク可視化         |

---

## 🧩 補足：企画工程の成果物の前後関係（流れ）

```
現状分析 → 課題整理 → システム化構想 → 効果・費用試算 → 実施体制 → 企画承認
```

成果物対応：

1️⃣ 現状分析報告書
2️⃣ 業務課題一覧
3️⃣ システム化方針書
4️⃣ ROI分析書
5️⃣ プロジェクト計画書
6️⃣ 稟議書・承認資料
