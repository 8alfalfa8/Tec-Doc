# ◆リソースバッファの設定

リソースバッファの設定目安について、詳しく説明します。キーパーソンの休暇・病気対応におけるリソースバッファは、プロジェクトのリスクマネジメントにおいて極めて重要です。

---

## 1. リソースバッファ設定の基本原則

### 考慮すべき要素
- **キーパーソンの重要性・代替難易度**
- **プロジェクト期間の長さ**
- **属人化の程度**
- **業務の複雑性**

## 2. 具体的な設定目安と計算方法

### 基本計算式
```
リソースバッファ = 基本バッファ + リスク調整バッファ
```

### 基本バッファ（標準的な状況）
**年間ベースでの目安：**
- **有給休暇対応**: 20〜25日/年（約8-10%）
- **病気休暇**: 5〜10日/年（約2-4%）
- **合計基本バッファ**: 25〜35日/年（約10-14%）

### プロジェクト期間別の換算
| プロジェクト期間 | 推奨バッファ率 | 具体例（1名あたり） |
|------------------|----------------|---------------------|
| **3ヶ月プロジェクト** | 3-4% | 2-3営業日 |
| **6ヶ月プロジェクト** | 6-7% | 7-9営業日 |
| **1年プロジェクト** | 10-14% | 25-35営業日 |
| **2年プロジェクト** | 20-25% | 50-65営業日 |

## 3. リスク要因に基づく調整係数

### 属人化度合いによる調整
| 属人化レベル | 調整係数 | 説明 |
|-------------|---------|------|
| **低（高い代替性）** | ×0.5 | 複数人が同スキル保有 |
| **中（限定的代替）** | ×1.0 | 部分的に代替可能 |
| **高（ほぼ不可欠）** | ×1.5-2.0 | 特定スキルが1名のみ |

### プロジェクトフェーズによる調整
- **設計・計画フェーズ**: ×1.2（クリティカル）
- **開発フェーズ**: ×1.0（標準）
- **テスト・保守フェーズ**: ×0.8（比較的柔軟）

## 4. 実践的な設定例

### ケーススタディ：中規模プロジェクト
```
プロジェクト期間: 6ヶ月
キーパーソン: リードエンジニア（属人化レベル: 高）
基本バッファ: 7日
調整係数: 1.5（属人化高い）
最終バッファ: 7日 × 1.5 = 10.5日 → 11日
```

### 複数キーパーソンがいる場合
```
チーム構成:
- プロジェクトマネージャ（属人化: 高）
- リードエンジニア（属人化: 高） 
- サブエンジニア（属人化: 中）

計算:
PM: 7日 × 1.5 = 11日
リード: 7日 × 1.5 = 11日
サブ: 7日 × 1.0 = 7日
総バッファ: 29日
```

## 5. バッファ管理のベストプラクティス

### 予防的対策との組み合わせ
- **ナレッジ共有**: ドキュメント化の徹底
- **クロストレーニング**: スキルの重複化
- **段階的引継ぎ**: 突然の不在に備える

### バッファ使用のルール明確化
```markdown
1. **事前承認不要**
   - 有給休暇（計画済み）
   - 病気休暇（2日以内）

2. **事前承認必要**
   - 長期休暇（3日以上）
   - 複数キーパーソン同時不在

3. **エスカレーション対象**
   - バッファ超過の見込み
   - クリティカルパスへの影響
```

## 6. モニタリングと調整

### 定期的な見直し指標
- **バッファ消費率**: 計画 vs 実績
- **休暇パターン分析**: 季節的な偏りの把握
- **病気休暇の傾向**: 過去実績に基づく調整

### ダッシュボード例
```
キーパーソンバッファ状況（2024年1月-3月）
```
| リソース | 計画残日数 | 実績残日数 | 利用率 |
|-----------|---------|---------|---------|
| リードEng | 10日 | 7日 | 30%  |
| アーキテクト | 8日 | 5日 | 38%  |
| デザイン責任者 | 6日 | 4日 | 33%  |

## 7. 組織規模による調整

### 小規模チーム（5名未満）
- **バッファ率**: 15-20%
- **重点管理**: 全員がキーパーソン扱い
- **対策**: 外部リソースの確保

### 中規模チーム（5-15名）
- **バッファ率**: 10-15%
- **重点管理**: コアメンバーのみ
- **対策**: 内部での代替要員確保

### 大規模チーム（15名以上）
- **バッファ率**: 8-12%
- **重点管理**: マネージャ層・専門家
- **対策**: 組織横断的なバックアップ

## 8. 法的・制度的考慮事項

### 法定休暇の確保
- **年次有給休暇**: 法定日数の確保
- **病気休暇**: 就業規則に基づく対応
- **育児・介護休暇**: 法定期間の考慮

### 業界特有の考慮点
- **繁忙期の考慮**: 業界特有のピーク考慮
- **オンコール対応**: 24/7体制の必要な場合
- **国際チーム**: 時間帯・祝日の違い

## 9. ツールを使った管理

### 推奨管理方法
- **リソース管理ツール**: Float, Resource Guru
- **プロジェクト管理ツール**: Jira, Asana
- **カレンダー連携**: 休暇計画の可視化

リソースバッファは「保険」のようなものです。過剰なバッファは非効率ですが、不足はプロジェクトリスクを高めます。過去の実績データを蓄積し、自組織に最適なバッファ率を見極めることが重要です。
