# 各工程のRACI表

以下に **SAP開発プロジェクトの標準工程に対応した RACI表** を、
**SIer／金融・公共系案件でそのまま使える実務粒度**で整理します。

---

## 前提：RACIの定義（共通認識）
[🔙 目次に戻る](#index)


| 区分                 | 意味           |
| ------------------ | ------------ |
| **R**（Responsible） | 実行責任者（手を動かす） |
| **A**（Accountable） | 最終責任者（決裁・承認） |
| **C**（Consulted）   | 相談・助言（双方向）   |
| **I**（Informed）    | 共有のみ（一方向）    |

## SAPプロジェクトRACIの重要ポイント（試験・監査で突っ込まれる点）
[🔙 目次に戻る](#index)


1. **業務がAを持たない工程は危険**
2. **Fit & Gap／UAT／移行は業務Aが必須**
3. **PMは「承認者」ではなく統括責任**
4. **QAは設計〜テストでRになる工程がある**
