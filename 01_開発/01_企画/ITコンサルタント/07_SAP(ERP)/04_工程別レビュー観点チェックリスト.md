# ◆ 工程別レビュー観点チェックリスト

以下に **SAP開発プロジェクトにおける工程別レビュー観点チェックリスト** を、
**金融・公共系／SIerレビュー・監査でそのまま使用できる粒度**で整理します。

> 目的：
>
> * 「何をレビューすべきか」を工程ごとに明確化
> * レビュー抜け・属人化を防止
> * 監査・第三者レビュー対応

---

# 0. 全体共通レビュー観点（全工程必須）

| 観点       | チェック内容               |
| -------- | -------------------- |
| スコープ     | 工程外作業が混入していないか       |
| トレーサビリティ | 上流成果物との対応関係が取れているか   |
| 承認       | A（Accountable）が明確か   |
| 標準遵守     | SAP標準・設計原則に違反していないか  |
| 将来影響     | 運用・改修・移行への影響を考慮しているか |

---

## ① 企画・準備工程 レビュー観点

| 観点   | チェック内容            |
| ---- | ----------------- |
| 導入目的 | 業務課題と導入目的が一致しているか |
| スコープ | 対象業務・対象外業務が明確か    |
| 標準方針 | 標準優先方針が明記されているか   |
| 体制   | 業務A／PM責任が明確か      |
| 前提条件 | 制約条件が明示されているか     |

---

## ② 要件定義工程 レビュー観点

| 観点    | チェック内容             |
| ----- | ------------------ |
| As-Is | 現行業務が正確に記載されているか   |
| To-Be | 将来業務の方向性が示されているか   |
| 業務粒度  | 要件が実装レベルに落ちすぎていないか |
| 非機能   | 性能・セキュリティ・監査要件があるか |
| 用語    | SAP用語に置き換えすぎていないか  |

---

## ③ Fit & Gap分析 レビュー観点（最重要）

| 観点     | チェック内容                |
| ------ | --------------------- |
| 判定根拠   | Fit/GAPの理由が明記されているか   |
| 標準理解   | 標準機能の正しい理解に基づいているか    |
| GAP最小化 | 業務変更の検討が行われたか         |
| 拡張方式   | Exit/BAdI/API使用方針が妥当か |
| 承認     | 業務部門がGAPを承認しているか      |

⚠ **ここで甘いと全工程破綻**

---

## ④ 基本設計（To-Be設計）レビュー観点

| 観点    | チェック内容             |
| ----- | ------------------ |
| 業務整合  | To-Be業務が要件と一致しているか |
| 組織設計  | 会社コード／プラント構成が妥当か   |
| マスタ設計 | 将来拡張・統廃合を考慮しているか   |
| 会計影響  | 仕訳・決算影響が整理されているか   |
| IF概要  | IF方式が明確か（同期／非同期）   |

---

## ⑤ 詳細設計（設定／ABAP）レビュー観点

### 設定（IMG）設計

| 観点   | チェック内容            |
| ---- | ----------------- |
| 設定一覧 | IMG項目が網羅されているか    |
| 設定理由 | なぜその値なのか説明できるか    |
| 影響範囲 | 他モジュール影響が整理されているか |
| 再現性  | 設定手順が明確か          |

---

### ABAP設計

| 観点     | チェック内容          |
| ------ | --------------- |
| 標準拡張   | 標準改修を行っていないか    |
| Exit使用 | Exit/BAdIが適切か   |
| DB設計   | HANA最適化されているか   |
| 例外処理   | 異常系・エラーメッセージ考慮  |
| 権限     | 権限チェックが実装されているか |

---

## ⑥ 開発・設定 レビュー観点

| 観点        | チェック内容            |
| --------- | ----------------- |
| 設計遵守      | 詳細設計通り実装されているか    |
| 標準非破壊     | 標準オブジェクトを変更していないか |
| 可読性       | ABAPコーディング規約遵守    |
| Transport | 移送ルールが守られているか     |
| 再実行性      | 再処理可能な実装か         |

---

## ⑦ 単体テスト レビュー観点

| 観点    | チェック内容         |
| ----- | -------------- |
| ケース網羅 | 正常／異常／境界値があるか  |
| 金額処理  | 桁・丸め誤差を確認しているか |
| 日付処理  | 月跨ぎ・期跨ぎ確認      |
| エラー   | メッセージが業務理解可能か  |
| 証跡    | 実行ログが残っているか    |

---

## ⑧ 結合テスト レビュー観点

| 観点      | チェック内容       |
| ------- | ------------ |
| シナリオ    | 実業務シナリオに基づくか |
| モジュール連携 | FI/MM/SD連動確認 |
| IF      | 送受信データ一致     |
| 会計整合    | 仕訳整合性        |
| 巻戻し     | 取消・再処理確認     |

---

## ⑨ UAT（業務受入）レビュー観点

| 観点  | チェック内容    |
| --- | --------- |
| 主体  | 業務部門が主体か  |
| データ | 本番相当データ使用 |
| 手順  | 実業務手順通りか  |
| 承認  | 業務Aによる承認  |
| 課題  | 残課題の扱い明確か |

---

## ⑩ 移行・切替 レビュー観点

| 観点   | チェック内容     |
| ---- | ---------- |
| 移行範囲 | 対象データが明確か  |
| 手順   | 再実行可能か     |
| リハ   | 実施済みか      |
| 突合   | 移行前後データ一致  |
| 切戻し  | ロールバック手順有無 |

---

## ⑪ 本番稼働・安定化 レビュー観点

| 観点   | チェック内容     |
| ---- | ---------- |
| 初期障害 | 想定済みか      |
| 監視   | ジョブ・IF監視   |
| 権限   | 本番権限確認     |
| 業務支援 | 問合せ体制      |
| 稼働判定 | Go/No-Go基準 |

---

## ⑫ 運用保守 レビュー観点

| 観点   | チェック内容    |
| ---- | --------- |
| 運用手順 | 属人化していないか |
| 変更管理 | 影響分析手順    |
| 権限管理 | 定期棚卸      |
| 監査   | ログ保持      |
| 改善   | 継続改善ループ   |

---

## SAPレビューでよくあるNGパターン

* Fit & Gapが「雰囲気」
* GAP理由が「標準で無理だから」
* 会計影響が後工程で発覚
* UATがITテスト化
* 移行手順が一発勝負
