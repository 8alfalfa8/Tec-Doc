# ◆ファンクションポイント法の適用範囲

ファンクションポイント法の工数見積もり結果について、適用範囲を整理します。

---

## 1. ファンクションポイント法の適用範囲

### 標準的な適用範囲
**ファンクションポイント法で見積もられる工数は、基本的に「開発工程」が中心です。**

```
全工程の内訳:
✅ 開発工程（FP法の主な適用範囲）
❌ 前工程（要件定義など）
❌ 後工程（導入・移行など）
```

## 2. 各工程の詳細な内訳

### FP法でカバーされる工程
```markdown
## 開発工程（FP法適用範囲）

### 1. 基本設計
- 外部設計
- データベース設計
- インターフェース設計

### 2. 詳細設計
- 内部設計
- プログラム設計
- テーブル設計

### 3. コーディング
- プログラミング
- 単体テスト作成
- コードレビュー

### 4. テスト
- 単体テスト
- 結合テスト
- システムテスト
```

### FP法でカバーされない工程
```markdown
## 非開発工程（別途見積もり必要）

### 1. 要件定義工程
- ヒアリング
- 要件分析
- 要件定義書作成

### 2. 導入・移行工程
- 環境構築
- データ移行
- ユーザー教育

### 3. プロジェクト管理
- 進捗管理
- 会議
- 報告書作成
```

## 3. 実践的な工数配分モデル

### 標準的な工程別工数配分
| 工程 | 工数比率 | FP法との関係 |
|------|----------|--------------|
| **要件定義** | 15-20% | 別途見積もり |
| **基本設計** | 10-15% | FP法範囲内 |
| **詳細設計** | 15-20% | FP法範囲内 |
| **コーディング** | 20-25% | FP法範囲内 |
| **テスト** | 20-25% | FP法範囲内 |
| **導入・移行** | 5-10% | 別途見積もり |
| **プロジェクト管理** | 10-15% | 別途見積もり |

### 計算例：100FPのプロジェクト
```markdown
前提条件:
- 生産性: 1.0人時/FP
- FP法工数: 100人時（開発工程のみ）

全体工数計算:
開発工程（FP法範囲）: 100人時（50%）
要件定義: 100 × 0.3 = 30人時（15%）
導入・移行: 100 × 0.15 = 15人時（7.5%）
プロジェクト管理: 100 × 0.25 = 25人時（12.5%）

総工数 = 100 + 30 + 15 + 25 = 170人時
```

## 4. 非開発工程の見積もり手法

### 要件定義工数の見積もり
```markdown
**機能ポイントベース**
要件定義工数 = FP数 × 0.2-0.3人時

**プロジェクト規模ベース**
- 小規模（〜50FP）: 10-20人時
- 中規模（51-200FP）: 20-50人時  
- 大規模（201FP〜）: 50-100人時
```

### 導入・移行工数の見積もり
```markdown
**考慮要素:**
- 環境構築の複雑さ
- データ移行量
- ユーザー数
- 教育トレーニング範囲

**目安:**
導入工数 = 開発工数 × 0.1-0.2
```

### プロジェクト管理工数の見積もり
```markdown
**標準モデル:**
プロジェクト管理工数 = 総開発工数 × 0.15-0.25

**変動要素:**
- チーム規模
- プロジェクト期間
- ステークホルダー数
- リスクレベル
```

## 5. 包括的な見積もりフレームワーク

### 総合工数計算式
```
総工数 = FP開発工数 + 非開発工数

FP開発工数 = 調整後FP × 生産性係数
非開発工数 = 要件定義 + 導入移行 + プロジェクト管理
```

### 詳細計算テンプレート
```markdown
## 総合工数見積もり書

**基本情報**
- プロジェクト名: XXXシステム開発
- 調整後FP: 120FP
- 適用生産性: 1.1人時/FP

**工数内訳**

1. FPベース開発工数
   - 120 FP × 1.1人時/FP = 132人時

2. 非開発工数
   - 要件定義: 132 × 0.2 = 26人時
   - 導入移行: 132 × 0.15 = 20人時
   - プロジェクト管理: 132 × 0.2 = 26人時
   - 非開発合計: 72人時

3. 総工数
   - 開発工数: 132人時
   - 非開発工数: 72人時
   - **総計: 204人時**
```

## 7. 実践的な適用アドバイス

### 組織カスタマイズの重要性
- **過去プロジェクトデータの分析**: 自組織の実績ベース配分を確立
- **プロジェクト特性の考慮**: 新規開発 vs 改修で配分変更
- **チーム成熟度の反映**: 経験値による効率化係数の導入

### 段階的な見積もり精緻化
```markdown
**初期見積もり（提案時）**
- FP法 + 標準配分比率

**詳細見積もり（要件定義後）**
- FP法 + プロジェクト特有の非開発工数見積もり

**最終見積もり（設計完了後）**
- 詳細FP計測 + 精确な非開発工数
```

## 8. よくある質問と回答

### Q: FP法だけでは不十分ですか？
**A:** はい。FP法は開発工数のみを計測します。全体の見積もりには非開発工程の追加見積もりが必要です。

### Q: 非開発工程の比率は固定ですか？
**A:** いいえ。プロジェクトの特性によって変わります。
- 新規開発: 要件定義工数が増加
- 既存システム改修: 影響調査工数が追加
- パッケージ導入: 導入工数が増加

## まとめ

ファンクションポイント法は**開発工程の工数見積もりに特化**した手法です。プロジェクト全体の正確な見積もりには、非開発工程を別途見積もる必要があります。

組織の実績データを蓄積し、プロジェクト特性に応じた調整を行うことで、より精度の高い見積もりが可能になります。

ご自身の組織の状況に合わせて、これらの比率を調整してみてください。
