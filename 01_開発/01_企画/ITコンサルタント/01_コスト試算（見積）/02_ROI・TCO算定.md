# ◆ ROI/TCO算定

以下では、**DX案件／基幹刷新案件におけるROI / TCO算定**を、
「**実務でどう作るか」「どう説明・合意するか**」まで含めて詳しく整理します。
（※経営会議・投資審議でそのまま使えるレベルです）

---

## 1. ROI / TCOの位置づけ（ITコンサル視点）

### なぜ重要か

* 投資判断の**定量根拠**
* DX／刷新が「費用」ではなく「経営投資」であることの説明
* 優先順位・スコープ決定の基準

---

## 2. TCO（Total Cost of Ownership）算定

### 2.1 目的

* システムの**ライフサイクル全体コスト**を把握
* 現行（As-Is）と将来（To-Be）のコスト比較

---

### 2.2 TCOに含めるコスト項目

#### 初期費用（CapEx）

| 項目      | 内容            |
| ------- | ------------- |
| 要件定義・設計 | コンサル、SI費      |
| 開発      | カスタマイズ、テスト    |
| インフラ    | HW / クラウド初期構築 |
| データ移行   | 移行ツール、作業      |
| 教育      | ユーザー教育        |

---

#### 運用費用（OpEx）

| 項目    | 内容         |
| ----- | ---------- |
| 保守    | ベンダー保守     |
| 運用    | 監視、障害対応    |
| 人件費   | 社内IT要員     |
| ライセンス | SaaS / ERP |
| インフラ  | クラウド利用料    |

---

#### 隠れコスト（重要）

* 障害対応工数
* 属人運用
* 手作業・Excel補完
* 法制度対応の突貫作業

---

### 2.3 算定期間

* 通常：**5年**
* 基幹刷新：7～10年もあり

---

### 2.4 TCO算定手順

#### Step1：現行TCO（As-Is）

* 過去実績＋ヒアリング
* 見えない工数を補正

#### Step2：将来TCO（To-Be）

* 標準運用前提
* 自動化効果反映

#### Step3：比較

```
TCO削減額 = As-Is TCO − To-Be TCO
```

---

### 2.5 成果物

* TCO算定表（年次）
* 前提条件一覧
* コスト削減内訳

---

## 3. ROI（Return on Investment）算定

---

### 3.1 目的

* 投資に対する**リターン可視化**
* 実行優先度・意思決定材料

---

### 3.2 ROI基本式

```
ROI (%) = (累積効果 − 投資額) ÷ 投資額 × 100
```

---

### 3.3 効果の種類

#### 定量効果（必須）

| 分類    | 例       |
| ----- | ------- |
| 人件費削減 | 工数削減    |
| 売上増   | 受注率向上   |
| 在庫削減  | 保管コスト削減 |
| 障害削減  | 損失回避    |

---

#### 定性効果（補完）

* 意思決定スピード
* 内部統制強化
* 従業員満足度

※定性は「定量に変換」できると強い

---

### 3.4 効果算定手順

#### Step1：KPI定義

* 業務時間
* 処理件数
* 在庫回転率

#### Step2：改善率設定

* 現場合意
* ベンチマーク

#### Step3：金額換算

```
削減時間 × 人件費単価
```

---

### 3.5 年次効果モデル

* 導入初年度：部分効果
* 定着後：100%効果

---

### 3.6 成果物

* ROI算定表
* 効果算定ロジック
* KPI対応表

---

## 4. DX案件でのROI/TCO算定例

---

### 例：受注DX

#### 投資

* 初期：1億円
* 運用：年間2,000万円

### 効果

| 項目   | 年間      |
| ---- | ------- |
| 工数削減 | 3,000万円 |
| 在庫削減 | 2,000万円 |
| 売上増  | 1,000万円 |

#### ROI（5年）

* 累積効果：3億円
* 投資：2億円
* ROI：+50%

---

## 5. 基幹刷新案件でのROI/TCO算定例

---

### 投資

* 初期：5億円
* 運用：年1億円

### 効果

* 保守費削減：年5,000万円
* 障害削減：年3,000万円
* 制度対応工数削減：年2,000万円

---

## 6. 経営説明でのポイント（超重要）

#### ① 前提条件を明示

* 人員削減するか
* 再配置するか

#### ② 数字は保守的に

* 過大評価はNG

#### ③ Before / After対比

* 図で説明

---

## 7. よくある失敗

| 失敗     | 対策      |
| ------ | ------- |
| 効果が抽象的 | KPI紐付け  |
| 初期費用だけ | TCOで説明  |
| 運用無視   | OpEx含める |

---

## 8. まとめ

> 「ROI/TCOは、IT投資を経営判断に耐える形で可視化し、
> 優先度とスコープを決めるための重要な分析です。」
