# ◆ファンクションポイント法

ファンクションポイント法（Function Point Analysis）について、詳細に説明します。これはソフトウェアの機能規模を測定し、工数見積もりに活用する標準的な手法です。

---
## 1. ファンクションポイント法の基本概念

### 概要と目的
- **機能規模の定量化**: ユーザー視点でのソフトウェア機能を数値化
- **技術非依存**: プログラミング言語や技術スタックに依存しない
- **早期見積もり**: 詳細設計前でも概算可能

### 基本プロセス
```
機能識別 → 分類 → 複雑度評価 → 未調整FP計算 → 調整係数適用 → 最終FP
```

## 2. 機能タイプの分類と識別

### 5つの基本機能タイプ

#### 1. 外部入力（EI: External Input）
**定義**: アプリケーション外部からデータを入力・更新する機能
**例**: 
- ユーザー登録フォーム
- 注文データ入力
- 在庫更新処理

#### 2. 外部出力（EO: External Output）
**定義**: アプリケーション外部へデータを出力する機能（処理含む）
**例**:
- 請求書発行
- 集計レポート
- グラフ表示

#### 3. 外部照会（EQ: External Query）
**定義**: データ検索・参照機能（処理なし）
**例**:
- 顧客情報検索
- 在庫照会
- 履歴参照

#### 4. 内部論理ファイル（ILF: Internal Logical File）
**定義**: アプリケーション内で管理するデータ群
**例**:
- 顧客マスタ
- 商品データベース
- 注文履歴ファイル

#### 5. 外部インターフェースファイル（EIF: External Interface File）
**定義**: 外部システムと共有する参照用データ
**例**:
- 外部決済システム連携
- 在庫管理システム参照
- 顧客CRMシステム連携

## 3. 複雑度評価と重み付け

### 複雑度決定要素

#### データ機能（ILF, EIF）の複雑度
| レコード要素数 | データ要素型数 | 複雑度 |
|---------------|---------------|--------|
| 1-19 | 1-4 | 低 |
| 20-50 | 5-15 | 中 |
| 51以上 | 16以上 | 高 |

#### トランザクション機能（EI, EO, EQ）の複雑度
| 参照ファイル数 | データ要素型数 | 複雑度 |
|---------------|---------------|--------|
| 0-1 | 1-4 | 低 |
| 2-3 | 5-15 | 中 |
| 4以上 | 16以上 | 高 |

### 重み付けテーブル（IFPUG標準）

| 機能タイプ | 低複雑度 | 中複雑度 | 高複雑度 |
|-----------|---------|---------|---------|
| **外部入力（EI）** | 3 | 4 | 6 |
| **外部出力（EO）** | 4 | 5 | 7 |
| **外部照会（EQ）** | 3 | 4 | 6 |
| **内部論理ファイル（ILF）** | 7 | 10 | 15 |
| **外部インターフェースファイル（EIF）** | 5 | 7 | 10 |

## 5. 調整係数（VAF: Value Adjustment Factor）

### 14のシステム特性

#### 計算式
```
VAF = (TDI × 0.01) + 0.65
ここで TDI = 14特性の合計点（0-70）
```

#### 14の特性と評価基準（0-5点）

1. **データ通信**（データ転送の程度）
2. **分散処理**（分散機能の程度）
3. **性能**（応答時間要件）
4. **使用負荷**（処理負荷の高さ）
5. **業務トランザクション率**（業務処理の頻度）
6. **オンラインデータ入力**（オンライン入力率）
7. **エンドユーザー効率**（ユーザーインターフェースの使いやすさ）
8. **オンライン更新**（オンライン更新の程度）
9. **複雑な処理**（処理ロジックの複雑さ）
10. **再利用性**（再利用可能性）
11. **インストールの容易さ**（導入・移行の容易さ）
12. **運用の容易さ**（運用管理のしやすさ）
13. **多拠点性**（複数拠点対応）
14. **変更容易性**（変更・拡張のしやすさ）

### VAF計算例
```
中程度のシステム特性を想定:
TDI = 14特性 × 平均3点 = 42点　　（※説明用の簡略化された例として、14× 平均3点）
VAF = (42 × 0.01) + 0.65 = 1.07
```

## 6. 調整後ファンクションポイント計算

### 計算式
```
調整後FP = 未調整FP × VAF
```

### 具体例
```
未調整FP: 51ポイント
VAF: 1.07
調整後FP = 51 × 1.07 = 54.57 → 55FP
```

## 7. 工数への変換

### 生産性基準の設定

#### 組織別生産性例（人時/FP）
| 組織成熟度 | 生産性 | 備考 |
|-----------|--------|------|
| **高成熟度** | 0.5-0.8人時/FP | CMMIレベル4-5 |
| **中成熟度** | 0.8-1.2人時/FP | 標準的な組織 |
| **低成熟度** | 1.2-2.0人時/FP | 経験浅い組織 |

### 工数計算例
```
調整後FP: 55FP
組織生産性: 1.0人時/FP（中成熟度）
総工数 = 55 × 1.0 = 55人時
```

### 期間への変換
```
1人月 = 160人時（20日×8時間）として
55人時 ÷ 160 = 0.34人月
```

## 8. 詳細な計算例：ECサイト開発

### 機能分析

| 機能コンポーネント | タイプ | 複雑度 | FP |
|-------------------|--------|--------|----|
| ユーザー登録 | EI | 中 | 4 |
| 商品検索 | EQ | 低 | 3 |
| カート追加 | EI | 低 | 3 |
| 注文確定 | EI | 高 | 6 |
| 決済処理 | EI | 高 | 6 |
| 注文履歴照会 | EQ | 中 | 4 |
| 在庫管理 | ILF | 高 | 15 |
| ユーザーマスタ | ILF | 中 | 10 |
| 決済システム連携 | EIF | 中 | 7 |
| **未調整FP合計** | | | **58** |


### 調整係数計算
```
TDI = 50点（ECサイトは高要件）
VAF = (50 × 0.01) + 0.65 = 1.15
調整後FP = 58 × 1.15 = 66.7 → 67FP
```

### 工数見積もり
```
生産性: 1.2人時/FP（Webシステム標準）
総工数 = 67 × 1.2 = 80.4人時
プロジェクト期間 = 80.4 ÷ 160 = 0.5人月
```

## 9. 実践的な適用ポイント

### 成功のためのコツ
- **過去プロジェクトデータの蓄積**: 自組織の生産性基準確立
- **一貫した計測方法**: 複数メンバーでの整合性確保
- **段階的な詳細化**: 初期見積もり後、随時精緻化

### よくある落とし穴
- **複雑度評価の主観性**: 評価基準の明確化が必要
- **調整係数の過大評価**: 客観的な評価の実施
- **生産性基準の誤り**: 実績データに基づく設定
