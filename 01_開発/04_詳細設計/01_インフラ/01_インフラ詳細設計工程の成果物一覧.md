# 🔧 インフラ詳細設計工程の成果物一覧

以下に、**システム開発における「詳細設計工程」でインフラチームが作成・担当する主な成果物一覧**を、カテゴリ別に整理します。
想定範囲は「サーバ／ネットワーク／クラウド／ミドルウェア／セキュリティ／監視／運用」などの要素を含む一般的なSI開発案件レベルです。

---

## 🔧 インフラ詳細設計工程の成果物一覧

### 1. サーバ／OS設計関連

| 分類                | 成果物名                                              | 内容概要 |
| ----------------- | ------------------------------------------------- | ---- |
| 物理／仮想構成設計書        | サーバ台数、役割、スペック（CPU・メモリ・ストレージ）、冗長構成、可用性要件を定義        |      |
| OS詳細設計書           | OSバージョン、パーティション構成、サービス設定、カーネルパラメータ、タイムゾーン・NTP設定など |      |
| サーバ命名規則・IPアドレス設計書 | サーバ名・ホスト名・IPアドレス体系、命名ルール、DNS設定方針                  |      |
| アカウント・権限設計書       | 管理者／運用アカウント、権限区分、sudoポリシー、パスワードポリシー               |      |

---

### 2. ネットワーク設計関連

| 分類              | 成果物名                                | 内容概要 |
| --------------- | ----------------------------------- | ---- |

| ファイアウォール／ACL設計書 | 通信許可／拒否ルール一覧、方向別通信設計、プロトコル指定        |      |

| ネットワーク接続図       | 各機器の接続関係、冗長経路、外部接続点の明示図             |      |

## 📘 補足

* 詳細設計では、「構成の確定」「設定パラメータの具体化」「構築手順の前段階」が目的です。
* 実装工程では、これらの設計書に基づいて**構築スクリプトやIaCコード（Terraform、Ansible）**が作成されます。
* 成果物はプロジェクトによって統合され、例えば「インフラ詳細設計書」に複数項目をまとめることもあります。
