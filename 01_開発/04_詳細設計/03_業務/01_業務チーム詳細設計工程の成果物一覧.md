# 詳細設計工程における業務チーム成果物一覧

以下は、**システム開発における「詳細設計工程」で業務チームが作成・担当する成果物一覧**の整理例です。
（前提：上流工程で基本設計が完了しており、機能単位・業務単位で詳細な処理設計やデータ仕様を定義するフェーズ）

---

## 🧭 1. 全体概要

| 区分     | 内容                                                     |
| ------ | ------------------------------------------------------ |
| 工程名    | 詳細設計（業務設計チーム）                                          |
| 主目的    | 基本設計で定義した業務要件・機能仕様をもとに、プログラムレベルでの処理仕様・データ仕様・画面仕様を具体化する |
| 成果物利用先 | 開発チーム（プログラマ／テストチーム）・品質保証チーム                            |

---

## 📘 2. 成果物一覧（業務チーム担当範囲）

| No | 成果物名              | 概要                   | 主な記載内容                            | 利用目的            |
| -- | ----------------- | -------------------- | --------------------------------- | --------------- |
| 1  | 詳細設計書（業務機能別）      | 各業務機能の処理仕様を詳細化       | 入出力定義、処理手順、分岐条件、例外処理、業務ルール        | 開発チームへ処理仕様伝達    |
| 2  | 画面設計書（UI詳細設計）     | 画面構成・項目定義・遷移・入力制御の定義 | 画面レイアウト、入力項目、バリデーション、遷移図、エラーメッセージ | UI開発、テストケース設計   |
| 3  | 帳票設計書             | 帳票出力の仕様詳細            | 帳票レイアウト、出力項目、集計・並び順、条件指定          | 帳票出力プログラム設計     |
| 4  | 業務ルール定義書          | ビジネスロジックや制御条件を明文化    | 計算式、判定条件、マスタ連携ロジック                | 処理共通ロジック定義      |
| 5  | データ項目定義書          | 各業務で使用するデータ項目の詳細     | データ型、桁数、必須/任意、変換ルール               | DB設計、I/F定義との整合  |
| 6  | 入出力インタフェース仕様書     | 外部システム・他モジュール連携の詳細仕様 | 入出力フォーマット、項目マッピング、通信方式、API仕様      | 開発・テスト時のI/F整合   |
| 7  | 業務フロー詳細図（シナリオ別）   | 業務プロセスをより詳細化         | 処理単位・画面操作・データ連携の流れ                | テストケースや教育資料への転用 |
| 8  | エラーハンドリング定義書      | 例外・異常系の動作仕様          | エラーコード、表示メッセージ、リカバリ処理             | 安定動作と品質確保       |
| 9  | コード体系定義書          | 各種コードや識別子の体系表        | コード値・意味・採番ルール                     | マスタ設計・画面制御共通化   |
| 10 | マスタ設定仕様書          | マスタデータの管理・初期値設定ルール   | マスタ項目、制御内容、登録手順                   | 環境構築、業務運用準備     |
| 11 | バッチ処理設計書          | 定期・一括処理の詳細仕様         | 入出力、スケジュール、処理手順、リトライ条件            | ジョブ設計・夜間処理制御    |
| 12 | テスト観点整理表（単体テスト支援） | 機能単位のテスト観点洗い出し       | 正常／異常パターン、入力組合せ、境界値               | 単体・結合テストケース作成支援 |

## 📂 3. 成果物管理（成果物間の関係）

```
業務フロー詳細図
   └─> 詳細設計書
         ├─> 画面設計書
         ├─> 帳票設計書
         ├─> バッチ処理設計書
         ├─> データ項目定義書
         └─> エラーハンドリング定義書
                └─> テスト観点整理表
```

---

## 🧩 4. チェック観点（品質確保のための確認項目）

| 区分       | チェック項目                    |
| -------- | ------------------------- |
| 整合性      | 基本設計との整合（機能ID、データ項目、画面項目） |
| 完整性      | 全機能に対して詳細設計書が作成されている      |
| 一貫性      | データ型・コード体系が他システム・DB設計と統一  |
| 明確性      | 処理ロジック・分岐条件・例外対応が具体的に記載   |
| トレーサビリティ | 要件→基本設計→詳細設計→テストの対応関係が明確  |

## 📑 5. 付属資料（必要に応じて業務チームが作成）

* 業務シナリオ別操作マニュアル（設計時点の準備資料）
* ダミーデータ仕様書（テスト環境投入用）
* 業務FAQ／レビュー指摘一覧
* ロジック例・擬似コード資料
