# PL／PM支援 業務範囲定義書（自治体システム標準準拠）

---

## 1. 文書目的
[🔙 目次に戻る](#index)


本書は、自治体システム標準準拠（以下「標準化」）プロジェクトにおいて、PL（Project Leader）およびPM（Project Manager）に対し提供する支援業務の範囲・役割・責任分界を明確化することを目的とする。

本定義書は、発注者（自治体）、受託事業者（SIer等）、PL／PM支援事業者の三者協働を前提とする。

## 8. 責任分界（RACI概念）
[🔙 目次に戻る](#index)


| 項目     | 自治体 | SIer | PL/PM支援 |
| ------ | --- | ---- | ------- |
| 意思決定   | R   | C    | A       |
| 設計・構築  | C   | R    | C       |
| 管理・統制  | C   | C    | R       |
| 標準準拠判断 | R   | C    | A       |

※R：責任、A：支援、C：協力

## 9. 留意事項
[🔙 目次に戻る](#index)


* 標準仕様・制度改定時は本定義を見直す
* プロジェクト規模に応じ支援粒度を調整可能
* フェーズ単位・部分支援契約も可

---

上記の定義書は特に次の点を意識しています。

* ✔ **発注者側支援（準委任）前提の立ち位置明確化**
* ✔ **標準仕様（Fit & Gap）をPL業務として明示**
* ✔ **PM＝統制、PL＝実行という役割分離**
* ✔ **SIerとの責任分界が揉めない構成**
* ✔ ガバメントクラウド／標準化遅延案件にも耐える内容
