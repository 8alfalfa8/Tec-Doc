# ◆ 自治体システム標準準拠における PL／PM支援

以下は **「自治体システム標準準拠における PL／PM支援」** を想定した、
**サービス説明（提案書・RFP回答・営業資料にそのまま使える粒度）** です。

---

## 1. 背景（自治体システム標準化・ガバメントクラウド）

自治体システムは以下を前提に進行しています。

* 地方公共団体情報システム標準仕様書（デジ庁）
* 標準準拠期限（令和7年度末）
* ガバメントクラウド（AWS / Azure / GCP / OCI）
* 標準化対象20業務＋関連周辺システム
* ベンダーロックイン排除／移行困難リスク

👉 **PL／PMには、従来のSI型開発管理を超えた統制力が要求される**

---

## 2. PL／PM支援の全体像（自治体向け）

### 役割定義

| 区分              | 役割                   |
| --------------- | -------------------- |
| PL（プロジェクトリーダ）   | 実行責任者・現場統制           |
| PM（プロジェクトマネージャ） | 全体統括・対外調整・品質保証       |
| 当社支援            | PL/PMの「右腕」として実務代行・補完 |

---

## 3. 支援スコープ（標準準拠特化）

### ① 標準仕様準拠マネジメント（最重要）

* 標準仕様書・適合基準の読み解き
* **Fit & Gap管理（標準 vs 既存）**
* 非標準機能の洗い出し／廃止判断支援
* 標準準拠チェックリスト作成・運用
* デジ庁QA・FAQ反映管理

---

### ② PL／PM業務代行・補佐

#### PMレベル

* 全体計画策定（WBS／マスタースケジュール）
* 進捗・課題・リスク管理（金融/公共向け粒度）
* ベンダーコントロール（複数社前提）
* 住民影響・業務影響の可視化
* 首長・情報政策課向け報告資料作成

#### PLレベル

* 要件定義～移行・テストまでの実行管理
* 業務部門ヒアリング同席・調整
* 設計レビュー（標準準拠観点）
* 移行リハーサル計画／切替統制

---

### ③ ガバメントクラウド対応支援

* クラウド方式選定支援（AWS/Azure/GCP/OCI）
* セキュリティ・運用（ISMAP前提）
* 標準AP × クラウド非機能要件整理
* 運用設計（自治体職員引継ぎ前提）

---

### ④ 移行・並行稼働リスク管理

* データ移行方式整理（マスタ／履歴）
* 旧システム並行稼働期間の運用整理
* 切替時の住民サービス影響最小化計画
* 障害・ロールバック計画

---

## 4. 成果物イメージ（例）

| 区分   | 成果物                 |
| ---- | ------------------- |
| 管理   | プロジェクト計画書、WBS、課題管理表 |
| 標準準拠 | Fit&Gap一覧、準拠チェックシート |
| 業務   | 業務影響整理資料、運用変更点一覧    |
| 移行   | 移行計画書、切替手順書         |
| 報告   | 定例報告資料（首長・幹部向け）     |

---

## 5. 他社PL/PM支援との差別化ポイント

* **「標準仕様 × 実務」両立**
* ベンダー側・発注者側 両視点
* 金融・公共向けの厳格な進捗／品質管理
* 技術・業務・運用を分断しない統合管理

---

## 6. よくある課題と解決

| 課題          | 支援内容           |
| ----------- | -------------- |
| 標準仕様が理解できない | 解釈・要点整理を代行     |
| ベンダー主導で進む   | 発注者側PL/PMとして統制 |
| 業務部門がついてこない | 業務影響を可視化       |
| 期限に間に合わない   | フェーズ再設計・優先度整理  |

---

## 7. 想定利用シーン

* 自治体情報政策課のPM補佐
* 元請SIのPL補完
* 複数ベンダー調整役（第三者PMO）
* 標準化遅延案件の立て直し
