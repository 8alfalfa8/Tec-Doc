# 📘 責任分界点一覧

以下に、**実務でそのまま使える「責任分界点一覧（RACI含む）」テンプレート＋記載例**を提示します。<br>
（オンプレ／クラウド／マネージドサービス混在を想定）

---

# 1. 目的

本書は、対象システムにおける各構成要素の責任範囲（責任分界点）を明確化し、
障害対応・変更管理・運用業務の役割を定義することを目的とする。

---

# 2. 責任分界点の考え方

責任は以下の区分で定義する。

| 区分      | 定義           |
| ------- | ------------ |
| 顧客      | システム利用者／業務部門 |
| SIer    | 設計・構築・運用委託先  |
| クラウド事業者 | IaaS/PaaS提供元 |
| DC事業者   | データセンター管理    |

# 4. 運用プロセス別責任分界点（RACI）

## RACI定義

| 記号 | 意味   |
| -- | ---- |
| R  | 実行責任 |
| A  | 最終責任 |
| C  | 相談   |
| I  | 報告   |

---

## 4.1 障害対応

| 作業      | 顧客 | SIer | クラウド |
| ------- | -- | ---- | ---- |
| 障害受付    | I  | R    |      |
| 一次対応    |    | R    |      |
| 二次解析    |    | R    | C    |
| ハード障害対応 |    |      | R    |
| 復旧確認    | A  | R    |      |

---

## 4.2 変更管理

| 作業   | 顧客 | SIer | クラウド |
| ---- | -- | ---- | ---- |
| 変更申請 | R  |      |      |
| 影響分析 | C  | R    |      |
| 実装   |    | R    |      |
| 承認   | A  |      |      |

---

# 5. クラウド利用時の責任モデル整理

IaaS利用時の一般モデル：

| レイヤ | 責任   |
| --- | ---- |
| 物理  | クラウド |
| 仮想化 | クラウド |
| OS  | 利用者  |
| MW  | 利用者  |
| アプリ | 利用者  |
| データ | 利用者  |

PaaS利用時はMW層までクラウド責任となる。

# 7. よくあるトラブル

| 問題     | 原因      |
| ------ | ------- |
| パッチ未適用 | OS責任未定義 |
| FW設定漏れ | 共同責任曖昧  |
| 障害長期化  | RACI未定義 |

---

# 8. レビュー観点

✔ レイヤごとに空白がないか<br>
✔ 共同責任が曖昧になっていないか<br>
✔ 障害対応責任が明確か<br>
✔ セキュリティ責任が定義済か<br>
✔ RACIでA（最終責任者）が1名か

---

# 🎯 実務での重要ポイント

* 「共同責任」は必ず詳細定義する
* 契約書・SLAと整合させる
* クラウド利用時は責任共有モデルを明文化する
* DR／バックアップの責任も必ず明記
