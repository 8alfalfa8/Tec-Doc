# 🧭 基本設計工程におけるインフラ領域の成果物一覧


以下に、**システム開発の基本設計工程（基本設計フェーズ）における「インフラ領域」の成果物一覧**を体系的に整理しました。
一般的なウォーターフォール型開発を前提としつつ、クラウド（AWS / Azure / GCP）利用を考慮した内容です。

---

## 🧭 基本設計工程におけるインフラ領域の成果物一覧

### 1. **全体構成設計関連**

| 成果物名             | 概要                                        | 備考                         |
| ---------------- | ----------------------------------------- | -------------------------- |
| システム構成図（インフラ構成図） | サーバ群、ネットワーク、ミドルウェア、クラウドサービスの構成を俯瞰する図。     | 論理構成／物理構成の2種類を用意する場合あり。    |
| 環境構成一覧           | 環境別（開発／検証／本番など）のサーバ、ネットワーク、クラウドサービス一覧。    | 各環境のIP、インスタンス種別、利用サービスを明記。 |
| 構成管理方針書          | 環境差分管理やIaC（Terraform, Ansible等）方針をまとめた資料。 | DevOps運用が前提の場合は必須。         |

---

### 2. **ネットワーク設計関連**

| 成果物名              | 概要                                      | 備考                         |
| ----------------- | --------------------------------------- | -------------------------- |
| ネットワーク基本設計書       | VLAN構成、サブネット設計、ルーティング、DNS、FWルールなどの全体設計。 | クラウドならVPC設計書に相当。           |
| 接続構成図             | 各サーバ／クラウドサービス間の通信経路を可視化。                | セキュリティレビューで利用。             |
| セキュリティグループ／ACL設計書 | 通信許可ポリシー（送信・受信ポート、CIDR）を一覧化。            | AWS SG, NACL, Azure NSGなど。 |
| ネットワークアドレス管理表     | サブネット／IPアドレス割当一覧。                       | DHCP・固定IPを明記。              |

---

### 3. **サーバ設計関連**

| 成果物名        | 概要                                  | 備考                                  |
| ----------- | ----------------------------------- | ----------------------------------- |
| サーバ基本設計書    | OS、CPU・メモリ・ストレージ構成、パッチ方針、冗長構成などを設計。 | クラウドならEC2/VM設定を含む。                  |
| サーバ一覧表      | サーバ名、用途、環境、スペック、IP等をまとめた一覧。         | 運用引継ぎにも使用。                          |
| ミドルウェア構成設計書 | Web/AP/DBなどのミドルウェア構成・バージョン・設定方針。    | Apache, Nginx, Tomcat, PostgreSQL等。 |
| ストレージ設計書    | ファイル配置、バックアップ領域、I/O要件など。            | S3, EBS, EFS, NFSなど。                |

---

### 4. **セキュリティ・監査関連**

| 成果物名           | 概要                          | 備考                                |
| -------------- | --------------------------- | --------------------------------- |
| セキュリティ設計書      | 認証・認可、暗号化、通信経路保護、監査ログなどの方針。 | ISMS・SOC2対応にも活用。                  |
| ログ管理設計書        | ログ取得、転送、保管、分析の設計。           | CloudWatch、SIEM（例：Splunk, ELK）連携。 |
| アクセス制御設計書      | 管理者・運用者のアクセス制御（IAM設計）。      | AWS IAM / Azure RBACなど。           |
| バックアップ・リストア設計書 | バックアップ対象、頻度、保存期間、復旧手順。      | DR対応含む。                           |

---

### 5. **可用性・運用設計関連**

| 成果物名          | 概要                                       | 備考                                       |
| ------------- | ---------------------------------------- | ---------------------------------------- |
| 冗長化設計書        | クラスタ構成、フェイルオーバー方式、ロードバランサ設計など。           | ALB/NLB、AutoScaling設定含む。                 |
| 監視設計書         | 監視項目、閾値、通知方法、ツール構成（Zabbix, CloudWatch等）。 | 運用監視チーム引継ぎに使用。                           |
| 運用設計書         | 運用体制、運用ツール、ジョブ管理、障害対応手順。                 | JP1, SystemWalker, AWS Systems Manager等。 |
| バッチ設計書（インフラ側） | バッチ実行サーバ、スケジューラ設定、リソース制御設計。              | cron, Step Functions, Data Pipelineなど。   |

---

### 6. **クラウド／IaC関連（クラウド利用の場合）**

| 成果物名           | 概要                                          | 備考                                  |
| -------------- | ------------------------------------------- | ----------------------------------- |
| クラウドアーキテクチャ設計書 | クラウドサービス選定理由、構成方針、冗長構成、コスト設計。               | AWS Well-Architected Framework準拠推奨。 |
| IaCテンプレート設計書   | Terraform/CloudFormationの設計方針、変数構成、モジュール分割。 | 開発～本番の環境分離を含む。                      |
| アカウント・課金設計書    | アカウント階層、請求管理、タグ運用ルール。                       | AWS Organizations等。                 |
| CI/CD設計書       | パイプライン構成（CodePipeline, GitHub Actionsなど）。   | IaC自動適用含む。                          |

## 🧩 補足：上流工程との関係

| 工程       | 主な目的                  | インフラ関連の成果物例                 |
| -------- | --------------------- | --------------------------- |
| 要件定義     | 性能・可用性・セキュリティ要件の明確化   | 非機能要件定義書、RTO/RPO要件書         |
| **基本設計** | 構成・方式・利用サービスを確定       | 上記成果物（構成図、設計書など）            |
| 詳細設計     | 設定値・パラメータ・スクリプトレベルで定義 | Terraform定義、OS設定表、監視閾値設定表など |
