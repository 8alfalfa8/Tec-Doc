# 🗺 システム構成図 L1（全体概要図）

## 🎯 目的

* システムの全体像を示す
* 外部接続関係を明確化
* 非機能設計の前提共有

---

## 📌 L1構成（概要）

```
           ┌───────────────────────┐
           │       利用者          │
           │  (社内/外部ユーザー)   │
           └──────────┬────────────┘
                      │ HTTPS
                      ▼
              ┌────────────────┐
              │   Route53      │
              └────────┬───────┘
                       ▼
              ┌────────────────┐
              │      ALB       │
              └────────┬───────┘
                       ▼
              ┌────────────────┐
              │   Web/AP層     │
              │   (ECS)        │
              └────────┬───────┘
                       ▼
              ┌────────────────┐
              │      DB層      │
              │     (RDS)      │
              └────────────────┘

      ────────────── 監視・運用系 ──────────────
      CloudWatch / CloudTrail / Backup / IAM
```

---

## 🔎 L1レビュー観点

* 外部公開範囲は適切か
* SPOFが存在しないか
* 監視系が含まれているか
* DR記載有無

---

# 🗺 システム構成図 L2（論理詳細図）

## 🎯 目的

* AZ構成
* サブネット分離
* 通信経路明確化
* セキュリティ境界可視化

---

## 📌 L2構成（論理）

```
VPC (10.0.0.0/16)
│
├── Public Subnet-A (AZ-a)
│     ├─ ALB
│     └─ NAT Gateway
│
├── Public Subnet-C (AZ-c)
│     ├─ ALB(冗長)
│     └─ NAT Gateway
│
├── Private Subnet-App-A
│     └─ ECS Task
│
├── Private Subnet-App-C
│     └─ ECS Task
│
├── Private Subnet-DB-A
│     └─ RDS (Primary)
│
└── Private Subnet-DB-C
      └─ RDS (Standby)

外部：
- S3（バックアップ）
- CloudWatch（監視）
- IAM（認証制御）
```

---

# 📊 通信フロー（L2補足）

| No | 通信元      | 通信先        | ポート  | 備考     |
| -- | -------- | ---------- | ---- | ------ |
| 1  | Internet | ALB        | 443  | HTTPS  |
| 2  | ALB      | ECS        | 443  | 内部通信   |
| 3  | ECS      | RDS        | 5432 | DB接続   |
| 4  | ECS      | S3         | 443  | バックアップ |
| 5  | 全体       | CloudWatch | 443  | ログ送信   |

---

# 🛡 セキュリティ境界（重要）

* Public層：ALBのみ
* App層：Private
* DB層：Private＋直接アクセス禁止
* Bastion経由管理（必要に応じ）

---

# 🎯 L1とL2の違い（実務整理）

| 項目       | L1   | L2   |
| -------- | ---- | ---- |
| 目的       | 全体共有 | 設計確定 |
| AZ表現     | 省略可  | 必須   |
| サブネット    | 省略可  | 必須   |
| 通信ポート    | 省略   | 明示   |
| セキュリティ境界 | 概要   | 明確化  |

---

# 📌 実務で評価が高い構成図の条件

✔ AZ分散が見える<br>
✔ Public/Privateが明確<br>
✔ 通信方向が矢印で明示<br>
✔ 監視・バックアップが図示<br>
✔ DR方式が記載

---

## 📌 L1テンプレート（汎用）

```
                  ┌────────────────────┐
                  │      利用者         │
                  │ (社内／外部／API)   │
                  └─────────┬──────────┘
                            │ HTTPS/API
                            ▼
                   ┌────────────────┐
                   │   DNS / CDN    │
                   └────────┬───────┘
                            ▼
                   ┌────────────────┐
                   │  WAF / FW層    │

                            ▼
                   ┌────────────────┐
                   │  LB（負荷分散） │
                   └────────┬───────┘
                            ▼
            ┌────────────────────────────────┐
            │        アプリケーション層        │
            │   (Web/AP/Batch/Container等)   │
            └──────────────┬─────────────────┘
                            ▼
                   ┌────────────────┐
                   │     DB層       │
                   │ (RDB/NoSQL等)  │
                   └────────┬───────┘

       監視基盤 / ログ管理 / IAM / Backup / DR
```
