# 📘 セキュリティ基本設計書（実務例）

以下に、**実案件レベル（クラウド＋オンプレ接続あり）のセキュリティ基本設計書 実務例**を提示します。<br>
想定：中〜大規模Webシステム／個人情報取扱あり／監査対象システム。

クラウド基盤は Amazon Web Services を利用する前提です。

# 1. 目的

本書は、EC販売管理システムにおけるセキュリティ対策の基本方針および設計内容を定義する。

---

# 2. セキュリティ設計方針

## 2.1 基本原則

* 最小権限の原則（Least Privilege）
* 多層防御（Defense in Depth）
* ゼロトラスト前提設計
* すべての通信を暗号化
* ログ取得前提設計（証跡重視）

---

## 2.2 対象範囲

* ネットワークセキュリティ
* サーバセキュリティ
* アプリケーションセキュリティ
* データ保護
* 認証認可
* ログ／監査
* インシデント対応

---

# 3. ネットワークセキュリティ設計

## 3.1 セグメント分離

| セグメント       | 内容      |
| ----------- | ------- |
| Public      | ALBのみ配置 |
| Private-App | APサーバ   |
| Private-DB  | DBサーバ   |
| 管理          | 踏み台サーバ  |

---

## 3.2 FW／SG設計方針

* 0.0.0.0/0許可は443のみ
* DBはAPセキュリティグループのみ許可
* 管理ポート（22/3389）は固定IP限定

---

## 3.3 WAF設計

* SQLインジェクション対策
* XSS対策
* Bot対策
* レート制限

---

# 4. 認証・認可設計

## 4.1 管理者認証

| 項目        | 内容    |
| --------- | ----- |
| 認証方式      | MFA必須 |
| パスワードポリシー | 12桁以上 |
| 有効期限      | 90日   |

---

## 4.2 アプリ利用者認証

| 項目      | 内容          |
| ------- | ----------- |
| 認証方式    | ID/PW＋二段階認証 |
| ロックアウト  | 5回失敗        |
| セッション管理 | 30分無操作      |

---

## 4.3 権限設計

* RBAC方式採用
* 管理者／一般／参照専用分離
* DBユーザもRO/RW分離

---

# 5. サーバセキュリティ設計

## 5.1 OS設定

* rootログイン禁止
* SSH鍵認証のみ
* 不要サービス停止
* CISベンチマーク準拠

---

## 5.2 パッチ管理

| 項目    | 内容     |
| ----- | ------ |
| 定期パッチ | 月次     |
| 緊急パッチ | 24時間以内 |
| 適用前検証 | 検証環境実施 |

---

## 5.3 EDR／ウイルス対策

* リアルタイム監視
* 定義ファイル自動更新
* 異常検知時SOC通知

---

# 6. データ保護設計

## 6.1 暗号化

| 対象     | 方式       |
| ------ | -------- |
| 通信     | TLS1.2以上 |
| データ保存  | KMS暗号化   |
| バックアップ | 暗号化      |
| パスワード  | ハッシュ化    |

---

## 6.2 個人情報保護

* マスキング処理
* ログ出力禁止
* 参照制限

---

# 7. ログ・監査設計

## 7.1 取得ログ

| 区分    | 内容      |
| ----- | ------- |
| OSログ  | 認証／sudo |
| DBログ  | 接続／DDL  |
| アプリログ | 認証／更新   |
| WAFログ | 攻撃検知    |

---

## 7.2 保存方針

| 項目    | 内容        |
| ----- | --------- |
| 保存期間  | 1年        |
| 保存場所  | 集中ログ基盤    |
| 改ざん防止 | 書込専用ストレージ |

---

# 8. 脆弱性対策

| 項目     | 内容     |
| ------ | ------ |
| 定期診断   | 年1回    |
| 自動スキャン | 月次     |
| 重大脆弱性  | 7日以内対応 |

---

# 9. インシデント対応

## 9.1 検知

* 不正ログイン5回
* 不審IP検知
* IDS/EDRアラート

## 9.2 対応フロー

1. 検知
2. 一時遮断
3. 調査
4. 影響分析
5. 報告
6. 再発防止

| RPO      | 15分 |

# 12. リスク一覧

| ------ | ------- | ----- |

# 🎯 実務レビューで必ず見られるポイント

* 0.0.0.0/0過剰許可がないか
* rootログイン禁止か
* MFA必須か
* 暗号化は通信／保存／バックアップすべて対応か
* ログ保存期間は監査要件満たすか
* インシデント対応フロー明文化されているか
