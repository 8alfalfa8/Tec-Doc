# 📘 DB構成基本設計書（実務例）

以下に、**実案件レベル（中〜大規模Webシステム／クラウド構成）のDB構成基本設計書 実務例**を提示します。<br>
想定：ECサイト本番環境、可用性99.95%以上、ピーク同時接続800。

基盤は Amazon Web Services を利用し、DBは Amazon RDS for MySQL を採用する。

# 1. 目的

本書は、本システムにおけるデータベース構成、可用性、性能、バックアップおよびDR設計を定義する。

# 2. 設計方針

## 2.1 基本方針

* Multi-AZによる高可用性
* 自動バックアップによるポイントインタイムリカバリ
* 3年後データ容量2倍を想定
* 原則暗号化
* アプリ側コネクションプール前提設計

# 🎯 実務レビューで必ず見られるポイント

* Multi-AZか単一AZか
* RTO/RPOが業務要件と整合しているか
* max_connectionsはAP側設定と整合しているか
* バックアップ保持期間は監査要件満たすか
* 暗号化は通信／保存／バックアップすべて対応か
* 復元テスト計画があるか
