# 📘 ネットワーク基本設計書（実務例）

以下に、**クラウド（AWS想定）＋オンプレ接続ありの実務レベル記載例**を示します。<br>
レビュー・監査にそのまま出せる粒度で記載しています。

※クラウド基盤は Amazon Web Services を前提とします。

---

# 1. 概要

## 1.1 目的

本書は、販売管理システム（以下、本システム）の本番環境におけるネットワーク構成および通信制御方針を定義する。

## 1.2 適用範囲

* AWS本番環境（ap-northeast-1）
* オンプレミス本社DCとの閉域接続
* 外部決済API接続

---

# 2. 設計方針

## 2.1 基本方針

* Public/Private完全分離
* 3層構成（LB / AP / DB）
* 最小権限通信
* マルチAZ冗長
* 将来拡張を考慮したCIDR設計（30%余裕）

## 2.2 非機能要件

| 区分     | 要件         | 対応           |
| ------ | ---------- | ------------ |
| 可用性    | 99.95%以上   | Multi-AZ構成   |
| セキュリティ | 外部直接DB接続禁止 | Private DB配置 |
| 拡張性    | 3年以内2倍増    | /16確保        |

---

# 🎯 実務レビューで必ず指摘される点

* NATが単一AZになっていないか
* DBがPublicにいないか
* 0.0.0.0/0がAP/DBに許可されていないか
* 通信一覧とSGが一致しているか
* オンプレCIDR重複していないか
