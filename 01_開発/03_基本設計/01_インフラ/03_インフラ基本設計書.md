# 2. 設計方針

## 2.1 アーキテクチャ方針

* クラウド：Amazon Web Services（東京リージョン）
* マルチAZ構成
* スケールアウト型設計
* Infrastructure as Code前提
* ゼロトラスト原則採用

---

## 2.2 非機能要件との対応

| 要件区分   | 要件        | 設計対応            |
| ------ | --------- | --------------- |
| 可用性    | 99.95%以上  | ALB＋Multi-AZ    |
| 性能     | 同時3,000接続 | ECS AutoScaling |
| セキュリティ | 通信暗号化     | TLS1.2以上        |
| バックアップ | RPO 15分   | 自動スナップショット      |
| 災害対策   | RTO 4時間   | 別AZ冗長           |

# 3. 全体構成

## 3.1 システム構成概要

### 構成要素

* Route53
* ALB
* ECS（Fargate）
* RDS（PostgreSQL）
* ElastiCache（Redis）
* S3
* CloudWatch
* IAM

---

## 3.2 論理構成図（L2レベル）

```
Internet
   ↓
Route53
   ↓
ALB（Public Subnet）
   ↓
ECS（Private Subnet）
   ↓
RDS（Private Subnet）
   ↓
Backup (S3)
```
