# 🧭 基本設計工程における業務チームの成果物一覧

以下は **「システム開発の基本設計工程において、業務チーム（業務担当・業務設計担当）が作成・関与する主な成果物一覧」** です。
一般的なウォーターフォール型開発を前提にしていますが、アジャイルでも一部は共通します。

---

## 🧭 基本設計工程における業務チームの主な役割

* 要件定義で確定した**業務要件・機能要件**をもとに、
  システムがどのように業務を支援するかを具体化する。
* ユーザー目線で**業務フロー・画面遷移・帳票出力・操作手順**を明確にする。
* 開発チームに対して、業務上の背景・ルール・例外処理を説明する。

---

## 📄 業務チームの成果物一覧（基本設計工程）

| 区分                | 成果物名                | 主な内容                        | 関与度          |
| ----------------- | ------------------- | --------------------------- | ------------ |
| **1. 業務設計関連**     | 業務フロー図（As-Is／To-Be） | 現行業務と新業務の流れ、業務分担、システム支援範囲   | 主担当          |
|                   | 業務機能構成図             | 業務単位・機能単位の構成と関連             | 主担当          |
|                   | 業務ルール定義書            | 入力・出力条件、計算式、処理ロジック、例外条件     | 主担当          |
|                   | 業務イベント一覧            | 業務トリガー（例：申請、承認、締め処理など）と発生条件 | 主担当          |
| **2. 画面・操作関連**    | 画面一覧・画面遷移図          | 画面間の遷移関係、処理単位の整理            | 主担当（開発と連携）   |
|                   | 画面レイアウト（ワイヤーフレーム）   | 画面項目、配置、入力ルール、表示仕様          | 主担当（開発とレビュー） |
|                   | 画面項目定義書             | 各項目の名称、データ型、桁数、入力制御、補足説明    | 主担当          |
| **3. 帳票・出力関連**    | 帳票一覧表               | 帳票の種類、用途、出力タイミング            | 主担当          |
|                   | 帳票レイアウト設計書          | 項目名、配置、集計ロジック、フォーマット仕様      | 主担当          |
| **4. データ関連**      | 業務データ一覧（概念レベル）      | 業務で扱うデータとその意味、関係性           | 共同担当（開発と連携）  |
|                   | コード・マスタ一覧           | コード体系、マスタ管理方法               | 主担当          |
| **5. インターフェース関連** | 業務間I/F一覧            | 他システム・部署とのデータ連携、トリガー、形式     | 共同担当         |
|                   | 入出力仕様書（I/F仕様書）      | 入出力データ項目、条件、頻度、ファイル形式       | 共同担当（開発主導）   |
| **6. 運用・管理関連**    | 業務カレンダー             | 稼働日、締め処理、月次・年次業務スケジュール      | 主担当          |
|                   | 業務権限一覧              | ユーザー区分と機能権限マッピング            | 主担当          |
|                   | エラー・例外対応方針          | 業務的なエラー発生時の取扱方針             | 主担当（開発と協議）   |
| **7. トレーサビリティ**   | 要件対応マトリクス（要件⇔機能⇔画面） | 各業務要件がどの機能で満たされるか整理         | 主担当（開発と連携）   |

## 📘 成果物の依存関係イメージ

```
業務要件定義書
   ↓
業務フロー図／機能構成図
   ↓
画面一覧・帳票一覧
   ↓
画面項目定義／帳票レイアウト設計
   ↓
I/F一覧／データ一覧
   ↓
権限一覧・運用カレンダー
```

---

## ✅ 補足（成果物レビュー観点）

業務チームは開発チームからの設計成果物（画面設計書、I/F仕様書、ER図など）に対しても以下の観点でレビューします。

* 業務ルールが正しく反映されているか
* 操作手順・画面遷移が実務に沿っているか
* 業務処理の抜け漏れや二重登録がないか
* 帳票・通知タイミングが業務サイクルに合っているか
