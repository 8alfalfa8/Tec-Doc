# ◆ 移行計画

以下では、**ITコンサルタント／移行リードの立場**で
**システム移行計画**（**例：オンプレ → クラウド、基幹刷新、データセンタ移設 等**）における
**「移行計画作業」フェーズ**を、**実務レベル**まで落として体系的に整理します。

---

## 1. 移行計画作業の位置づけ
[🔙 目次に戻る](#index)


| フェーズ          | 目的                   |
| ------------- | -------------------- |
| 現行調査・分析       | 移行対象と制約を把握           |
| **移行計画（本作業）** | **「いつ・何を・どう移すか」を確定** |
| 移行設計          | 技術的な実装設計             |
| 移行実施          | 本番移行作業               |
| 移行後対応         | 安定化・最適化              |

👉 **移行計画は「技術 × 業務 × 経営判断」を束ねる中核作業**です。

---

## 2. 移行計画作業の全体タスク構造
[🔙 目次に戻る](#index)


```
移行計画作業
├─ ① 移行対象・スコープ定義
├─ ② 移行方式選定（戦略決定）
├─ ③ 移行単位・順序設計
├─ ④ データ移行計画
├─ ⑤ 切替方式・リリース計画
├─ ⑥ 体制・役割・RACI定義
├─ ⑦ スケジュール・マイルストーン策定
├─ ⑧ リスク・課題・制約整理
├─ ⑨ 移行コスト・工数見積
└─ ⑩ 移行計画書作成・合意
```

以下、**各タスクを詳細に説明**します。
