# ◆ 在庫管理システムの業務知識

以下は、**在庫管理システムの業務知識**を、**業務観点・システム観点の両面**から体系的に整理した内容です。
（※小売・製造・卸に共通する標準モデル＋実務でよく問題になる点も含めています）

---

## 1. 在庫管理システムの目的と役割

### 目的

* **欠品防止**（機会損失の防止）
* **過剰在庫抑制**（キャッシュフロー・保管コスト削減）
* **在庫可視化**（数量・金額・場所・状態）
* **業務効率化・自動化**
* **会計・監査対応**（棚卸・評価）

### 管理対象

* 商品（SKU）
* 原材料
* 半製品（仕掛品）
* 製品
* 消耗品

---

## 2. 在庫管理の基本業務プロセス（全体像）

```
① 発注
② 入荷
③ 保管・ロケーション管理
④ 引当
⑤ 出庫（出荷・払出）
⑥ 返品・廃棄
⑦ 棚卸
⑧ 在庫評価・分析
```

---

## 3. 業務別 詳細業務知識

### ① 発注管理（購買）

#### 主な業務

* 発注点管理（定量・定期）
* 発注量計算
* 発注書作成
* 納期管理

#### 発注方式

| 方式     | 概要               |
| ------ | ---------------- |
| 定量発注方式 | 在庫が発注点を下回ると一定量発注 |
| 定期発注方式 | 一定周期で必要量を発注      |
| MRP    | 需要予測・生産計画連動      |
| JIT    | 必要な時に必要な量        |

#### システム機能

* 発注点・安全在庫設定
* 自動発注提案
* サプライヤマスタ管理

---

### ② 入荷管理

#### 主な業務

* 入荷予定確認
* 現品照合（数量・品質）
* 入荷登録
* 不良品処理

#### ポイント

* **検品結果による在庫区分**

  * 良品
  * 不良品
  * 保留品

#### システム機能

* 入荷予定 vs 実績差異管理
* バーコード／QRスキャン
* ロット・期限管理

---

### ③ 保管・ロケーション管理

#### 管理軸

* 倉庫
* エリア
* 棚
* 段・間口

#### 在庫区分

* 良品在庫
* 返品在庫
* 不動在庫
* デッドストック

#### システム機能

* ロケーションマスタ
* 空棚管理
* FIFO / FEFO制御

---

### ④ 引当管理

#### 引当とは

> 受注・生産に対して「使う予定の在庫」を確保すること

#### 種類

* 受注引当
* 生産引当
* 仮引当／本引当

#### システム機能

* 在庫引当ロジック
* 引当優先順位制御
* 引当残・未引当管理

---

### ⑤ 出庫（出荷・払出）管理

#### 主な業務

* 出庫指示
* ピッキング
* 出庫検品
* 出庫確定

#### 出庫形態

| 形態   | 例         |
| ---- | --------- |
| 販売出庫 | 顧客向け出荷    |
| 生産払出 | 製造ラインへの供給 |
| 移動出庫 | 倉庫間移動     |

#### システム機能

* ピッキングリスト
* ハンディターミナル連携
* 出庫差異管理

---

### ⑥ 返品・廃棄管理

#### 返品理由

* 誤出荷
* 不良品
* 期限切れ

#### システム機能

* 返品受付
* 再利用可否判定
* 廃棄記録（監査対応）

---

### ⑦ 棚卸業務

#### 種類

| 種類   | 概要        |
| ---- | --------- |
| 実地棚卸 | 現物カウント    |
| 循環棚卸 | 一部を定期的に棚卸 |

#### システム機能

* 棚卸差異算出
* 調整仕訳連携
* 履歴管理

---

### ⑧ 在庫評価・分析

#### 在庫評価方法

* 先入先出法（FIFO）
* 移動平均法
* 総平均法

#### 代表的KPI

| 指標     | 意味      |
| ------ | ------- |
| 在庫回転率  | 在庫効率    |
| 在庫回転日数 | 滞留度     |
| 欠品率    | サービスレベル |
| 滞留在庫率  | 不良在庫把握  |

---

| 倉庫マスタ     | 倉庫・エリア    |

## 6. 実務でよくある課題

* 理論在庫と実在庫の乖離
* 引当ミスによる欠品
* ロット・期限管理不備
* 属人化（Excel管理）
* システムと現場運用の不整合

# ◆ 在庫管理 業務フロー図（全体）

以下に、**在庫管理システムの詳細業務フロー図**（**標準・実務レベル**）を
**業務単位 × 判断ポイント × システム処理**まで落として整理します。
（※小売／卸／製造で共通利用できるベースフローです）

---

## 在庫管理 詳細業務フロー図（全体）

```
【需要発生】
   │
   ├─① 発注計画／補充判断
   │
   ├─② 発注
   │
   ├─③ 入荷
   │
   ├─④ 保管・ロケーション管理
   │
   ├─⑤ 引当
   │
   ├─⑥ 出庫（出荷／払出）
   │
   ├─⑦ 返品・廃棄
   │
   ├─⑧ 棚卸
   │
   └─⑨ 在庫評価・分析
```

---

### ① 発注計画・補充判断（起点）

```
［販売実績／需要予測］
        │
        ▼
［現在庫確認］
        │
        ▼
［在庫量 <= 発注点？］──No──→ 終了
        │Yes
        ▼
［発注量算出］
        │
        ▼
［発注可否判断］
```

#### 業務内容

* 在庫数量確認
* 発注点・安全在庫チェック
* 発注量計算

#### システム処理

* 自動発注提案
* 発注アラート
* 需要予測連携（任意）

---

### ② 発注業務

```
［発注内容確定］
        │
        ▼
［発注書作成］
        │
        ▼
［仕入先送信］
        │
        ▼
［入荷予定登録］
```

#### 管理項目

* 商品
* 数量
* 納期
* 単価
* 仕入先

---

### ③ 入荷業務（検品含む）

```
［入荷予定あり］
        │
        ▼
［現品到着］
        │
        ▼
［数量検品］
        │
        ▼
［品質検品］
        │
        ├─良品──→ 在庫計上
        │
        ├─不良品→ 不良在庫登録
        │
        └─保留品→ 保留在庫登録
```

#### 判断ポイント

* 数量差異
* 品質不良
* ロット・期限相違

---

### ④ 保管・ロケーション管理

```
［入荷確定］
        │
        ▼
［保管場所決定］
        │
        ▼
［ロケーション登録］
        │
        ▼
［在庫更新］
```

#### 管理軸

* 倉庫
* エリア
* 棚番
* ロット
* 使用期限

---

### ⑤ 引当業務（非常に重要）

```
［受注／生産指示］
        │
        ▼
［在庫存在？］──No──→ 欠品処理
        │Yes
        ▼
［引当実行］
        │
        ▼
［引当済在庫］
```

#### 引当種類

* 仮引当
* 本引当
* 部分引当

---

### ⑥ 出庫（出荷／払出）業務

```
［出庫指示］
        │
        ▼
［ピッキング］
        │
        ▼
［出庫検品］
        │
        ├─OK──→ 出庫確定
        │
        └─NG──→ 差異処理
```

#### 出庫種別

* 販売出庫
* 生産払出
* 倉庫移動

---

### ⑦ 返品・廃棄業務

```
［返品発生］
        │
        ▼
［状態判定］
        │
        ├─再利用可→ 良品在庫
        │
        ├─条件付→ 保留在庫
        │
        └─不可──→ 廃棄処理
```

#### 管理ポイント

* 理由コード
* 廃棄承認
* 証跡管理

---

### ⑧ 棚卸業務

```
［棚卸指示］
        │
        ▼
［実地カウント］
        │
        ▼
［理論在庫比較］
        │
        ├─一致──→ OK
        │
        └─差異──→ 調整処理
```

#### 差異原因

* 入出庫未登録
* 破損
* 誤ピッキング

---

### ⑨ 在庫評価・分析

```
［在庫データ］

        ▼

        ▼
```

#### 分析内容

# ◆ 在庫管理システム構成図（WMS含む）

以下に、**① 在庫管理システム構成図**（**WMS含む**） を、
**業務・システム・現場の責務分離が分かるレベル**で整理します。

（※小売／卸／製造で共通利用でき、金融・公共案件でも通用する粒度です）

---

## 全体アーキテクチャ（論理構成）

```
┌─────────────────────────────────────────────────────┐
│          利用者（業務）                              │
│  購買 / 販売 / 生産 / 倉庫                           │
└──────────────┬──────────────────────────────────────┘
               │ Web / API
               ▼
┌─────────────────────────────────────────────────────┐
│        基幹業務システム層                             │
│                                                     │
│  ┌──────────────────────────────────────────┐       │
│  │ 在庫管理システム（基幹）                   │       │
│  │ ・理論在庫                                │       │
│  │ ・在庫引当                                │       │
│  │ ・在庫区分                                │       │
│  │ ・在庫評価                                │       │
│  └──────────────────────────────────────────┘       │
│        ▲          ▲          ▲          ▲           │
│        │          │          │          │           │
│ ┌──────┘   ┌──────┘   ┌──────┘   ┌──────┘           │
│ │          │          │          │                  │
│ ▼          ▼          ▼          ▼                  │
│購買管理   販売管理   生産管理  会計システム            │
│                      (MRP)   （在庫評価・仕訳・監査） │
│                                                     │
│                                                     │
└──────────────┬──────────────────────────────────────┘
               │ 指示／実績連携
               ▼
┌─────────────────────────────────────────────────────┐
│        WMS（倉庫管理システム）                        │
│  ・物理在庫管理（ロケーション）                        │
│  ・入荷／出庫／移動指示                                │
│  ・作業実績管理                                       │
└──────────────┬───────────────────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────────────────┐
│        倉庫現場オペレーション                          │
│  ハンディ端末 / RFID / 自動倉庫                        │
└──────────────────────────────────────────────────────┘
```

---

## システムごとの役割分担（重要）

| レイヤ      | 主責務                |
| -------- | ------------------ |
| 在庫管理（基幹） | **論理在庫・引当・会計連携**   |
| WMS      | **物理在庫・作業指示・実績管理** |
| 現場       | **実作業（動かす・数える）**   |

👉 **「在庫数を決める」のは在庫管理システム**
👉 **「どこから取るか・どう動くか」はWMS**

---

## 在庫データの考え方（よくある設計ポイント）

```
理論在庫（基幹） ≠ 物理在庫（WMS）
        │
        └─ 定期的・イベント駆動で同期
```

* 即時同期：出庫・入庫
* 非即時：棚卸・移動中在庫

---

# ◆ WMS連携前提の物理作業フロー（詳細）

以下に、**② WMS連携前提の物理作業フロー（現場作業）** を、
**業務・システム・現場の責務分離が分かるレベル**で整理します。

（※小売／卸／製造で共通利用でき、金融・公共案件でも通用する粒度です）

---

## 1️⃣ 入荷作業フロー（WMS主導）

```
［基幹：入荷予定登録］
        │
        ▼
［WMS：入荷指示生成］
        │
        ▼
［現場：荷受け］
        │
        ▼
［検品（数量・品質）］
        │
        ├─OK──→ 仮置き
        │
        └─NG──→ 不良エリア
        │
        ▼
［棚入れ指示（WMS）］
        │
        ▼
［指定ロケーションへ格納］
        │
        ▼
［入荷実績確定］
        │
        ▼
［基幹：在庫反映］
```

### ポイント

* **ロケーション決定はWMS**
* 基幹は「入った事実」を管理

---

## 2️⃣ 保管・ロケーション管理フロー

```
［在庫発生］
    │
    ▼
［WMS：最適ロケーション選定］
    │
    ▼
［現場：棚入れ］
    │
    ▼
［ロケーション在庫更新］
```

### 制御要素

* FIFO / FEFO
* 温度帯
* 重量制限
* 動線最適化

---

## 3️⃣ 引当〜出庫フロー（基幹 × WMS 連携の核心）

```
［受注／生産指示］
        │
        ▼
［基幹：在庫引当］
        │
        ▼
［WMS：出庫指示生成］
        │
        ▼
［現場：ピッキング］
        │
        ▼
［出庫検品］
        │
        ▼
［出庫実績確定］
        │
        ▼
［基幹：在庫減算・売上/払出反映］
```

### 超重要ポイント

* **引当は基幹**
* **ピッキング順・経路はWMS**

---

## 4️⃣ 倉庫内移動フロー

```
［移動指示］
    │
    ▼
［WMS：移動元・先指示］
    │
    ▼
［現場：移動作業］
    │
    ▼
［移動実績登録］
```

👉 基幹は「場所」までは持たないケースが多い

---

## 5️⃣ 返品・廃棄フロー

```
［返品受付］
    │
    ▼
［WMS：検品指示］
    │
    ▼
［状態判定］
    │
    ├─良品→ 通常在庫
    ├─保留→ 保留ロケーション
    └─廃棄→ 廃棄エリア
    │
    ▼
［基幹：在庫区分反映］
```

---

## 6️⃣ 棚卸フロー（WMS主導）

```
［基幹：棚卸指示］
        │
        ▼
［WMS：棚卸リスト生成］
        │
        ▼
［現場：実地棚卸］
        │
        ▼
［差異算出（WMS）］
        │
        ▼
［基幹：差異調整・会計反映］
```

---

## 基幹 × WMS 連携インターフェース例

| 連携種別  | 内容        |
| ----- | --------- |
| マスタ連携 | 商品・倉庫     |
| 指示連携  | 入荷・出庫・移動  |
| 実績連携  | 入荷実績・出庫実績 |
| 例外連携  | 差異・不良・廃棄  |

---

## よくある設計ミス（注意）

❌ 在庫数をWMSと基幹で二重管理<br>
❌ 引当をWMSで実施<br>
❌ 現場例外を基幹に直接入力<br>
❌ ロケーションを基幹で細かく管理
