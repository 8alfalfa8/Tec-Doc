# ◆ DCM（Demand Chain Management）システムの業務知識

以下は、**DCM（Demand Chain Management）システムの業務知識**を、
**業務視点 × システム視点 × 小売・大企業向け実務レベル**で整理した内容です。
（※SCM知識がある前提で、一段深い実務寄りにしています）

---

## 1. DCMとは何か（SCMとの違い）

### ■ 定義

**DCM（Demand Chain Management）**とは、
**「需要（Demand）」を起点に、供給・生産・在庫・物流・販売を最適化する経営・業務・ITの仕組み**です。

| 観点   | SCM         | DCM          |
| ---- | ----------- | ------------ |
| 起点   | 供給（作れる量・在庫） | 需要（売れる量・顧客）  |
| 主眼   | 効率化・コスト削減   | 需要予測精度・売上最大化 |
| 時代背景 | 大量生産・安定需要   | 需要変動・短サイクル   |
| データ  | 過去実績中心      | 実績＋予測＋外部要因   |

👉 **DCMは「需要主導型SCM」**と理解すると実務で通じます。

---

## 2. DCMが対象とする業務領域（全体像）

```
需要予測
   ↓
需要計画（Demand Planning）
   ↓
供給計画（Supply Planning）
   ↓
生産計画／調達計画
   ↓
在庫・物流・販売
   ↓
実績フィードバック → 需要予測へ
```

DCMは**「計画系（Planning）」が主戦場**です。

---

## 3. DCMの主要業務知識（業務別）

---

### ① 需要予測（Demand Forecasting）

#### ■ 業務内容

* 過去販売実績分析
* トレンド／季節性分析
* キャンペーン影響考慮
* 外部要因反映（天候・経済・SNS等）

#### ■ 重要指標

* MAPE（平均絶対誤差率）
* Bias（過剰/過小予測）
* Forecast Accuracy

#### ■ 実務ポイント

* **「当たらない前提」で補正・合意形成が重要**
* 営業・マーケの経験値を数値に反映

---

### ② 需要計画（Demand Planning）

#### ■ 業務内容

* SKU×期間別需要計画作成
* チャネル別（EC/店舗/法人）配分
* 計画値の合意（営業・生産・物流）

#### ■ 成果物

* 月次/週次需要計画
* 需要計画確定版（Frozen）

#### ■ 実務ポイント

* **需要計画＝経営判断の翻訳**
* 営業主張 vs 生産制約の調整が核心

---

### ③ 供給計画（Supply Planning）

#### ■ 業務内容

* 生産能力・調達制約考慮
* 需要計画に対する供給可否判定
* 不足時の優先順位付け

#### ■ 代表概念

* ATP（Available To Promise）
* CTP（Capable To Promise）

#### ■ 実務ポイント

* 需給ギャップを「見える化」することが価値
* 100%充足より「損失最小化」

---

### ④ 生産・調達計画連携

#### ■ 業務内容

* MPS（基準生産計画）
* MRP（資材所要量計画）
* 調達LT・ロット考慮

#### ■ 実務ポイント

* DCMは**ERPの上位レイヤ**
* SAP / Oracle ERPとの連携必須

---

### ⑤ 在庫最適化

#### ■ 業務内容

* 適正在庫水準算定
* 安全在庫算定
* 滞留在庫／死蔵在庫対策

#### ■ 指標

* 在庫回転率
* 欠品率
* 廃棄率

---

### ⑥ 販売・S&OP（経営連携）

#### ■ S&OP（Sales & Operations Planning）

* 営業・生産・物流・経営の合意形成会議
* 月次意思決定プロセス

#### ■ DCMの役割

* 数字の「共通言語」を提供
* 感覚論を排除

---

## 4. DCMシステムの機能構成（IT視点）

### ■ 機能一覧

| 機能     | 内容        |
| ------ | --------- |
| 需要予測   | 統計／AI予測   |
| シナリオ分析 | What-if分析 |
| 需給調整   | ギャップ可視化   |
| S&OP支援 | 合意形成      |
| KPI管理  | 精度・在庫     |

---

### ■ 代表的なDCM製品

* SAP IBP
* Oracle Demantra
* Kinaxis RapidResponse
* Blue Yonder

---

## 5. DCM × AI／データ活用（近年トレンド）

### ■ AI活用例

* 機械学習による需要予測
* 異常検知（突発需要）
* 自動補正・自律型計画

### ■ データソース

* POS
* ECアクセス
* 天候API
* SNS分析

---

## 6. DCM導入時の実務的課題

| 課題         | 実務的な原因  |
| ---------- | ------- |
| 予測精度が上がらない | 業務データ品質 |
| 現場が使わない    | 合意形成不足  |
| ERP連携失敗    | マスタ不整合  |
| 効果が見えない    | KPI未定義  |

👉 **業務改革（BPR）なしのDCM導入は失敗率が高い**

---

## 7. 小売・公共・大企業案件での位置づけ

* 基幹刷新（ERP更改）の**上位構想**
* DX文脈の「意思決定高度化」
* グループ横断需給可視化

# ◆ 小売業向け DCM 業務フロー図（詳細）

以下は **小売業向けDCM（Demand Chain Management）業務フロー図（詳細）** を、
**現場業務で使える粒度＋システム連携まで落とした形**で整理したものです。
（総合スーパー／専門店／EC併売を想定）

---

## 1. 全体俯瞰（エンドツーエンド）

```
① データ収集
   ↓
② 需要予測
   ↓
③ 需要計画
   ↓
④ S&OP（需給調整）
   ↓
⑤ 補充・発注計画
   ↓
⑥ 仕入・物流・在庫実行
   ↓
⑦ 販売・実績収集
   ↓
⑧ 分析・改善（予測へフィードバック）
```

---

## 2. 業務フロー詳細（業務 × システム）

---

## ① データ収集（Input）

### 業務内容

* POS販売実績取得
* EC販売データ取得
* 在庫残・入荷予定取得
* マスタ（商品／店舗／取引先）
* 外部データ（天候・イベント）

### システム連携

| データ | システム      |
| --- | --------- |
| POS | 店舗POS     |
| EC  | EC基盤      |
| 在庫  | WMS / ERP |
| 外部  | API       |

```
POS / EC / WMS / 外部データ
        ↓
   DCM データ基盤
```

---

## ② 需要予測（Demand Forecasting）

### 業務内容

* SKU × 店舗 × 日／週単位予測
* 季節性・曜日性考慮
* 特売・販促反映
* 異常値補正

### 実務ポイント（小売特有）

* 店舗ごとの売れ筋差が大きい
* 新商品は類似SKU補完

### 成果物

* ベース需要予測
* 予測精度レポート

```
過去実績 + 外部要因
        ↓
   需要予測エンジン
        ↓
   需要予測結果
```

---

## ③ 需要計画（Demand Planning）

### 業務内容

* 売場計画・販促計画反映
* 店舗別／チャネル別需要調整
* 部門責任者レビュー

### 関与部門

* 商品部
* 営業企画
* EC部門

### 成果物

* 確定需要計画（Frozen）

```
需要予測
   ↓（人手調整）
需要計画（確定）
```

---

## ④ S&OP（需給調整・経営判断）

### 業務内容

* 需要 vs 供給能力比較
* 欠品／過剰リスク算定
* 優先順位決定（売場・商品）

### 意思決定例

* 欠品覚悟で利益率重視
* 一部店舗制限販売

### 成果物

* 合意済需給計画

```
需要計画
   ↓        ↘
供給制約 → S&OP会議 → 調整済計画
```

---

## ⑤ 補充・発注計画（Replenishment）

### 業務内容

* 店舗別補充量算定
* DC（物流拠点）在庫考慮
* 発注点／安全在庫算定

### 発注方式

* 自動発注（定型商品）
* 手動発注（特売・高額品）

### 成果物

* 発注指示データ

```
需給計画
   ↓
補充・発注計算
   ↓
発注データ
```

---

## ⑥ 仕入・物流・在庫実行（Execution）

### 業務内容

* 取引先発注
* 入荷・検品
* 店舗配送
* 棚入れ

### システム

* ERP（発注）
* WMS（物流）
* TMS（配送）

```
発注 → 入荷 → DC保管 → 店舗配送 → 店舗在庫
```

---

## ⑦ 販売・実績収集（Sell）

### 業務内容

* 店舗／EC販売
* 欠品・廃棄記録
* 在庫変動

### 実績データ

* 実売数
* 値引・廃棄
* 欠品回数

```
販売実績
   ↓
DCM実績DB
```

---

## ⑧ 分析・改善（Feedback）

### 業務内容

* 予測誤差分析
* 店舗別乖離分析
* ルール改善

### KPI

| KPI   | 意味    |
| ----- | ----- |
| 欠品率   | 売機会損失 |
| 在庫回転率 | 資金効率  |
| 廃棄率   | ロス管理  |
| 予測精度  | 計画品質  |

```
実績分析
   ↓
予測・計画ロジック改善
   ↓
需要予測へ戻る
```

---

## 3. 小売業DCMの特徴まとめ

| 項目    | 小売特有ポイント      |
| ----- | ------------- |
| 粒度    | SKU×店舗が非常に細かい |
| 変動    | 天候・販促影響大      |
| 重要性   | 欠品＝即売上損失      |
| DCM価値 | 「売れる量を外さない」   |
