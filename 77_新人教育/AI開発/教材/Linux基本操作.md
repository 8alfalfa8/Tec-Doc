# ◆ Linux基本操作

Linux基本操作は、サーバーやクラウド、AI開発環境を扱う上で欠かせない基礎スキルです。Linuxはオープンソースのオペレーティングシステムであり、多くのWebサーバー、クラウド基盤、開発環境で標準的に利用されています。そのため、基本的な操作を理解することは、ITエンジニアにとって重要な第一歩となります。

主な操作はコマンドライン（CUI）で行われます。例えば、ファイル一覧の表示は「ls」、ディレクトリ移動は「cd」、ファイル作成は「touch」、コピーは「cp」、削除は「rm」などがあります。また、権限管理（chmod、chown）やプロセス管理（ps、kill）、パッケージ管理（apt、yum）といった概念も重要です。これらを理解することで、システムの構造や動作を深く把握できるようになります。

Linuxはスクリプト（シェルスクリプト）による自動化にも優れており、定型作業の効率化や運用の標準化に貢献します。さらに、DockerやKubernetesなどのコンテナ技術、クラウド環境（AWS、GCP、Azure）もLinuxを前提としているため、インフラからアプリケーションまで幅広い領域で活用されます。

近年のAI開発においても、Linuxの重要性はますます高まっています。多くのAIフレームワーク（[Python](Pythonプログラム開発（基礎編）.md)、PyTorch、TensorFlowなど）はLinux環境での利用が前提となっており、GPU制御や分散処理、データ処理基盤の構築にもLinuxの知識が不可欠です。また、AIモデルのデプロイや運用（MLOps）においても、Linux上での操作や自動化が中心となります。

このようにLinux基本操作は、単なるOS操作にとどまらず、インフラ、開発、AI運用を支える共通基盤です。AI時代においては、MarkdownやMermaid、Git等と並び、エンジニアが必ず身につけるべき基礎技術の一つといえます。

以下では、新人教育に最適なLinuxコマンドを、**「まずこれだけ覚えれば業務が始められる」** という視点で厳選して整理しました。

## 1.絶対に覚えるべき基本5コマンド

### pwd（現在地を確認）
```bash
pwd
# 例: /home/username/project
```
今自分がどこにいるか分からなくなったら、まずこれを打ちます。

### ls（ファイル一覧を見る）
```bash
ls                    # シンプルに一覧
ls -l                 # 詳細情報付き（ファイル権限、所有者、サイズ）
ls -a                 # 隠しファイル（.から始まるファイル）も表示
ls -la                # よく使う組み合わせ
```

### cd（移動する）
```bash
cd ディレクトリ名     # 指定したディレクトリに移動
cd ..                 # 1つ上のディレクトリに移動
cd ~                  # ホームディレクトリに移動
cd /                  # ルートディレクトリに移動
cd -                  # さっきまでいたディレクトリに戻る
```

_★★★絶対パスと相対パスについて要理解（重要）_

### mkdir（ディレクトリを作る）
```bash
mkdir ディレクトリ名
mkdir -p a/b/c        # 途中のディレクトリも一緒に作る
```

### touch（ファイルを作る）
```bash
touch ファイル名.txt
# 既にあるファイルの場合は更新日時が変わる
```

## 2.ファイル操作の基本

### cp（コピー）
```bash
cp 元ファイル 新しいファイル名
cp -r 元ディレクトリ 新しいディレクトリ名  # ディレクトリごとコピー
```

### mv（移動・リネーム）
```bash
mv ファイル名 新しい場所/
mv 古い名前 新しい名前  # リネームにも使う
```

### rm（削除）※注意して使う
```bash
rm ファイル名
rm -r ディレクトリ名   # ディレクトリを削除
rm -f ファイル名       # 確認なしで強制削除（要注意）
```
**rm -rf /** は絶対に実行しないでください（全て消えます）。

### cat（ファイルの中身を見る）
```bash
cat ファイル名
cat ファイル1 ファイル2  # 複数ファイルを連結して表示
```

## 3.テキスト編集・閲覧（現場でよく使う）

### less（長いファイルを1ページずつ見る）
```bash
less ファイル名
# 操作: スペース(次ページ)、b(前ページ)、q(終了)、/(検索)
```

### head / tail（ファイルの最初/最後だけ見る）
```bash
head -n 10 ファイル名  # 最初の10行
tail -n 20 ファイル名  # 最後の20行
tail -f ログファイル    # ログをリアルタイムで監視（超便利）
```

### grep（ファイルの中身を検索）
```bash
grep "検索文字" ファイル名
grep -r "検索文字" ディレクトリ  # ディレクトリ以下を再帰的に検索
grep -i "検索文字" ファイル名    # 大文字小文字区別しない
```

## 4.権限・プロセス関連

### chmod（実行権限を与える）
```bash
chmod +x スクリプト.sh  # 実行権限を追加
chmod 755 ファイル名    # 数字でも指定できるが、+xで十分な場合が多い
```

### ps（動いているプロセスを見る）
```bash
ps aux                 # 全ユーザーの全プロセスを表示
ps aux | grep キーワード  # 特定のプロセスを探す
```

### kill（プロセスを止める）
```bash
kill プロセスID
kill -9 プロセスID      # 強制終了（どうしても止まらない時）
```

## 5.ディスク・ファイル容量

### df（ディスクの空き容量）
```bash
df -h                  # 人間に読みやすい形式で表示
```

### du（ディレクトリの使用量）
```bash
du -sh ディレクトリ名   # そのディレクトリの合計サイズ
du -h --max-depth=1    # 1階層下までのサイズを表示
```

## 6.便利な小技（知っておくと楽）

### 履歴から再利用
```bash
history                # 今まで打ったコマンドの履歴
!123                   # 履歴番号123のコマンドを再実行
!!                     # 直前のコマンドを再実行
Ctrl + r               # 履歴をインクリメンタル検索
```

### ワイルドカード
```bash
ls *.txt               # .txtで終わる全ファイル
rm log-2024-*          # log-2024-で始まる全ファイルを削除
```

### リダイレクトとパイプ
```bash
command > ファイル名    # コマンドの結果をファイルに保存
command >> ファイル名   # 追記
command1 | command2    # command1の結果をcommand2に渡す
```

## 8.実務での1日の流れ

**朝、サーバーにログイン:**
```bash
pwd                    # 今どこ？
cd /var/www/project    # プロジェクトへ移動
ls -la                 # 何があるか確認
```

**ログを調べる:**
```bash
cd logs
ls -lt                 # 新しい順に表示
tail -f access.log     # リアルタイムでログ監視
grep "ERROR" error.log # エラーを探す
```

**ファイルを編集:**
```bash
cp config.php config.php.bak  # バックアップ取ってから
vi config.php          # 編集（viの使い方は別途）
```

## 9.注意：絶対にやってはいけないこと

1. **rm -rf /** - 全てのファイルが消えます
2. **chmod 777** - セキュリティ危険
3. **sudo よく分からないコマンド** - 権限が必要な時は必ず意味を確認

## 10.練習方法

```bash
# 練習用ディレクトリを作って、そこで試す
mkdir ~/linux_practice
cd ~/linux_practice
mkdir a b c
touch test.txt
# いろいろ試す
```

分からないコマンドは `man コマンド名` または `コマンド名 --help` でヘルプが見られます。エラーメッセージが出ても慌てず、メッセージを読む習慣をつけましょう。
