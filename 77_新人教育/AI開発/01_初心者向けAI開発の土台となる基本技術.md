# ◆ 初心者向けAI開発の土台となる基本技術

AI自律開発やAIDDに入る前に、**土台となる基本技術**を押さえることが非常に重要です。  
ここでは「初心者が実務に入れるレベル」までを意識して、体系的に整理します。

---

## ① なぜ基本技術が重要か

AIツール（例：GitHub Copilot や Cursor）は強力ですが、

👉 **前提知識がないと**

* 出力の良し悪しが判断できない
* バグを見抜けない
* セキュリティリスクを理解できない

つまり  
👉 **「AIを使いこなす力 = 基本技術 × AI活用力」**

---

## ② 最低限必要な基本技術（全体像）

初心者は以下の順番で学ぶのが効率的です：

1. Markdown（ドキュメント）
2. Git / GitHub（ソース管理）
3. Linuxコマンド
4. プログラミング（Python推奨）
5. Web基礎（HTTP / API）
6. データ基礎（JSON / DB）

---

## ③ Markdown（最優先）

### ■ Markdownとは

軽量な記法でドキュメントを書くための言語
（設計書・README・仕様書で必須）

👉 AI開発では「プロンプト管理」「設計書」に頻出

---

### ■ 基本記法

````markdown
# 見出し1
## 見出し2

- 箇条書き
- 箇条書き

1. 番号付きリスト
2. 番号付きリスト

**太字**
*斜体*

`コード`

```python
print("Hello")
```

[リンク](https://example.com)

````

---

### ■ 実務での使い方
- README.md（プロジェクト説明）
- 設計書（DocDD）
- API仕様書
- AIプロンプト管理

👉 **AIDDではMarkdownが設計書そのものになる**

---

## ④ Git / GitHub（必須）

### ■ ツール

- [GitHub公式ドキュメント](https://docs.github.com)

### ■ 基本概念
- リポジトリ（コード保管庫）
- コミット（変更履歴）
- ブランチ（作業分岐）
- マージ（統合）

---

### ■ 基本コマンド

```bash
git clone URL
git add .
git commit -m "message"
git push
````

---

### ■ なぜ重要か

* チーム開発の前提
* AI生成コードの管理
* 変更履歴の追跡

👉 **AI開発でもGitが中心**

---

## ⑤ Linuxコマンド（インフラ基礎）

### ■ よく使うコマンド

```bash
ls        # ファイル一覧
cd        # ディレクトリ移動
mkdir     # フォルダ作成
rm        # 削除
cat       # ファイル表示
grep      # 検索
```

---

### ■ なぜ必要か

* サーバー操作
* Docker / クラウド操作
* ログ調査

👉 インフラ・AI開発どちらでも必須

---

## ⑥ プログラミング（Python推奨）

### ■ なぜPythonか

* AIとの相性が良い
* ライブラリが豊富
* 学習コストが低い

---

### ■ 基本文法

```python
# 変数
x = 10

# 条件分岐
if x > 5:
    print("big")

# ループ
for i in range(3):
    print(i)

# 関数
def add(a, b):
    return a + b
```

---

### ■ AI開発で使う例

* API呼び出し
* データ処理
* エージェント実装

---

## ⑦ Web基礎（API理解）

### ■ 重要概念

### ■ なぜ重要か

* Python基礎

## ⑪ 重要な結論

👉 AI時代でも本質は変わらない

* Markdown → 思考整理
* Git → 変更管理
* Python → 実装
* API → 接続
* DB → データ

その上に  
👉 AI（Copilotなど）が乗る
