# ◆ ALB 同時接続数設計

以下では **ALB（Application Load Balancer）の同時接続数設計**について、
**オンプレミス Tomcat システムを ECS に移行する前提**で、

* ① 設計の全体像
* ② オンプレ値からの算出方法
* ③ ALB／ECS／Tomcat それぞれの制限
* ④ 実務での設計パターン
* ⑤ 失敗しやすいポイント

を **設計判断ができるレベル**まで掘り下げて説明します。

## 3. 同時接続数の基本計算（最重要）
[🔙 目次に戻る](#index)


### 3.1 Littleの法則（実務必須）
[🔙 目次に戻る](#index)


```
同時接続数 = RPS × 平均応答時間（秒）
```

#### 例（オンプレ）
[🔙 目次に戻る](#index)


```
RPS = 200
応答時間 = 0.4秒

→ 同時接続数 = 200 × 0.4 = 80
```

※ Keep-Alive 有効時は **×2〜5倍** を想定

```
80 × 3 = 240 接続
```

➡ **設計同時接続数 = 約250**
