# ◆ Fargate vs EC2の設計比較

以下では **ECS on Fargate と ECS on EC2 の設計比較**を、
**オンプレ Tomcat（WAR）→ AWS ECS 移行**という文脈で、

* 設計観点別の違い
* リソース設計・性能・制限
* コスト・運用・スケーリング
* Tomcat / Java ワークロードでの実務判断基準
* 典型的な選定パターン

まで **設計レビューにそのまま使えるレベル**で整理します。

| ----------- | --------- | ----------- |

| 運用負荷        | ◎（OS管理不要） | △（OS/AMI管理） |

| Java/Tomcat | ○         | ◎           |
| 長期大規模       | △         | ◎           |

## 2. アーキテクチャの違い
[🔙 目次に戻る](#index)


### 2.1 Fargate
[🔙 目次に戻る](#index)


```
ALB
 └─ ECS Service
     └─ Task（Fargate）
         └─ Container
```

* OS / EC2 不可視
* Task単位で課金
* CPU / Memory 組み合わせ固定

---

[🔙 目次に戻る](#index)


### 2.2 EC2
[🔙 目次に戻る](#index)


```
ALB
 └─ ECS Service
     └─ EC2 Instance
         └─ Docker
             └─ Task × 複数
```

* インスタンス管理あり
* リソース共有
* 細かいチューニング可能

## 4. Java / Tomcat 観点の違い
[🔙 目次に戻る](#index)


| 観点              | Fargate | EC2  |
| --------------- | ------- | ---- |
| JVM GC          | 制限あり    | 最適化可 |
| HugePage        | ❌       | ⭕    |
| swap            | ❌       | ⭕    |
| File descriptor | 固定      | 調整可  |
| Kernel param    | 不可      | 可能   |

👉 **低レイテンシ or 高スループットはEC2有利**
