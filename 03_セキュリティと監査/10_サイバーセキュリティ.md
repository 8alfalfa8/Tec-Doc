# ◆ サイバーセキュリティ

サイバーセキュリティに関わる**必須知識**と**実務での対策作業**を、
実務（金融・公共・大規模システム前提）レベルで体系的に整理します。

---

## 1. サイバーセキュリティの全体像

まず前提となる概念です。

### 【従来のセキュリティの三大原則（CIA）】

| 要素  | 英語              | 内容      | 具体例        |
| --- | --------------- | ------- | ---------- |
| 機密性 | Confidentiality | 不正閲覧を防ぐ | 暗号化、アクセス制御 |
| 完全性 | Integrity       | 改ざんを防ぐ  | ハッシュ、署名    |
| 可用性 | Availability    | 継続利用可能  | 冗長化、DDoS対策 |

---

### 【追加される4要素】

| 要素    | 英語              | 内容            | 実務での意味  |
| ----- | --------------- | ------------- | ------- |
| 真正性   | Authenticity    | 本物であることの保証    | なりすまし防止 |
| 責任追跡性 | Accountability  | 誰が行ったか追跡可能    | ログ監査    |
| 否認防止  | Non-repudiation | 実行者が後から否定できない | 電子署名    |
| 信頼性   | Reliability     | 正確かつ安定して動作    | 品質保証・監査 |

---

#### ① 機密性（Confidentiality）

##### ■ 必須知識

* 認証（Authentication）
* 認可（Authorization）
* 暗号技術（共通鍵・公開鍵）
* 通信暗号化（TLS）
* アクセス制御モデル（RBAC / ABAC）
* DLP（情報漏えい対策）
* 個人情報保護法・GDPR

##### ■ 対策作業

| 項目     | 作業内容                             |
| ------ | -------------------------------- |
| アクセス制御 | 最小権限設計、ロール設計                     |
| 暗号化    | データ保存時暗号化（AES-256）、通信暗号化（TLS1.3） |
| 認証強化   | 多要素認証（MFA）、SSO導入                 |
| 鍵管理    | KMS運用、鍵ローテーション                   |
| データ分類  | 機密区分定義（公開/社内/機密/極秘）              |

##### ■ 実装例

* AWS：IAM + KMS + S3暗号化
* オンプレ：Active Directory + BitLocker

---

#### ② 完全性（Integrity）

##### ■ 必須知識

* ハッシュ関数（SHA-256）
* デジタル署名
* 改ざん検知
* ログの不可逆保管（WORM）
* チェックサム

##### ■ 対策作業

| 項目      | 作業内容                           |
| ------- | ------------------------------ |
| ログ保全    | 監査ログの改ざん防止                     |
| ファイル監視  | FIM（File Integrity Monitoring） |
| バージョン管理 | Git管理・変更履歴保持                   |
| DB整合性   | 制約設計・トランザクション管理                |

##### ■ 実装例

* SIEM導入
* DB監査ログ有効化

---

#### ③ 可用性（Availability）

##### ■ 必須知識

* 冗長構成（HA）
* フェイルオーバー
* バックアップ設計
* DR（災害対策）
* DDoS対策

##### ■ 対策作業

| 項目     | 作業内容            |
| ------ | --------------- |
| 冗長化    | 負荷分散（LB）、クラスタ構成 |
| バックアップ | 日次/週次バックアップ     |
| 監視     | 24/365監視、アラート設計 |
| DR訓練   | 年1回以上の切替訓練      |

##### ■ 実装例

* AWS：Multi-AZ構成
* オンプレ：Active-Standby構成

---

#### ④ 真正性（Authenticity）

##### ■ 必須知識

* PKI（公開鍵基盤）
* 証明書管理
* なりすまし攻撃（Phishing / MITM）
* FIDO2 / WebAuthn

##### ■ 対策作業

| 項目    | 作業内容           |
| ----- | -------------- |
| 証明書管理 | 有効期限監視         |
| メール対策 | SPF/DKIM/DMARC |
| MFA   | 生体認証・ハードウェアキー  |

---

#### ⑤ 責任追跡性（Accountability）

##### ■ 必須知識

* 監査ログ
* 操作ログ設計
* SIEM
* 内部不正対策

##### ■ 対策作業

| 項目     | 作業内容          |
| ------ | ------------- |
| ログ取得   | OS/アプリ/DBログ統合 |
| 保存期間   | 1〜7年（業界基準）    |
| 定期レビュー | 月次監査          |

---

#### ⑥ 否認防止（Non-Repudiation）

##### ■ 必須知識

* 電子署名法
* タイムスタンプ
* デジタル証明書
* ブロックチェーン技術

##### ■ 対策作業

| 項目   | 作業内容      |
| ---- | --------- |
| 電子契約 | 電子署名導入    |
| 証跡管理 | タイムスタンプ付与 |
| 取引記録 | 改ざん不能保存   |

---

#### ⑦ 信頼性（Reliability / Safety）

##### ■ 必須知識

* Secure SDLC
* 脆弱性管理
* ペネトレーションテスト
* セキュリティ標準（ISO27001）

##### ■ 対策作業

| 項目      | 作業内容        |
| ------- | ----------- |
| 脆弱性診断   | 年1回以上       |
| パッチ管理   | 月次適用        |
| 教育訓練    | 年1回セキュリティ研修 |
| CSIRT設置 | インシデント対応体制  |

# ◆ サイバーハイジーン（Cyber Hygiene）

**サイバーハイジーン**（**Cyber Hygiene**）とは、

> 「高度な対策の前に、必ず継続的に実施すべき基本的防御習慣」

を意味します。

高度なSOCやゼロトラストよりも先に、まず徹底すべき“基礎体力”です。

---

## ■ サイバーハイジーンとは

米国では CISA（米国サイバーセキュリティ庁）や
NIST が継続的に強調しています。

特徴：

* 高度ではない
* 難しくない
* しかし「徹底が難しい」
* 実はインシデントの80%以上を防げる

---

## ■ 必須のサイバーハイジーン 10項目（実務レベル）

---

## ① 資産管理（Asset Management）

### なぜ重要か

守る対象が分からなければ守れない。

### 必須作業

* サーバ・端末・ネットワーク機器一覧化
* SaaS利用状況棚卸
* ソフトウェア一覧（SBOM）
* クラウドリソース棚卸

### 実務ポイント

* CMDB運用
* 月次更新
* 未承認機器の遮断（NAC）

---

## ② パッチ管理（Patch Management）

### 目的

既知脆弱性を塞ぐ

### 必須作業

* OS月次更新
* ミドルウェア更新
* 緊急パッチ（48時間以内）
* EOL製品排除

### 実務基準

* 重大（CVSS 9以上）→即時対応
* 重要→1週間以内

---

## ③ 強固な認証

### 必須作業

* 全管理者MFA必須
* 長期パスワード禁止
* 使い回し禁止
* 特権ID分離

### 重要概念

* ゼロトラスト
* 最小権限原則

---

## ④ アクセス制御（最小権限）

### 必須作業

* ロール定義
* 不要権限削除
* 退職者即時削除
* 四半期棚卸

---

## ⑤ バックアップ

### 目的

ランサムウェア対策

### 必須作業

* 3-2-1ルール
* オフライン保存
* 定期リストア試験

---

## ⑥ ログ監視

### 必須作業

* 認証ログ監視
* 管理者操作監査
* 異常通信検知
* 保存期間1年以上（業界による）

---

## ⑦ マルウェア対策

### 必須作業

* EDR導入
* ウイルス対策更新
* USB制御
* マクロ制限

---

## ⑧ メールセキュリティ

### 必須作業

* SPF / DKIM / DMARC設定
* 添付ファイル検査
* URLサンドボックス

---

## ⑨ ネットワーク分離

### 必須作業

* 本番／検証分離
* 社内／外部公開分離
* 管理系ネットワーク隔離

---

## ⑩ セキュリティ教育

### 必須作業

* 年1回以上の教育
* 標的型メール訓練
* インシデント報告手順周知

---

## ■ 技術別チェックリスト（具体）

| 要素    | ハイジーン例 |
| ----- | ------ |
| 機密性   | アクセス制御 |
| 完全性   | パッチ管理  |

---

## ■ 実務での失敗パターン

* パッチは方針だけ存在
* 退職者IDが放置
* 管理者MFA未実装
* バックアップ未検証
* ログ保存しているだけで未監視

高度なSOCよりも、まずここが崩れます。

## ■ 結論

サイバーハイジーンとは

> 「高度な防御ではなく、基本を徹底する文化」

です。

多くの大規模事故は、
ゼロデイではなく「既知脆弱性放置」から発生します。
