## 3. DNSSEC（Domain Name System Security Extensions：DNSセキュリティ拡張）設計
[🔙 目次に戻る](#index)


### 3.1 DNSSEC概要
[🔙 目次に戻る](#index)


DNSSECはDNS応答に電子署名を付与し、応答の真正性・完全性を保証する。

[🔙 目次に戻る](#index)


### 3.2 鍵構成設計
[🔙 目次に戻る](#index)


| 鍵種別 | 役割     | 鍵長            | ローテーション |
| --- | ------ | ------------- | ------- |
| KSK | 親ゾーン登録 | RSA 2048      | 1–2年    |
| ZSK | レコード署名 | RSA 1024/2048 | 3–6か月   |

[🔙 目次に戻る](#index)


### 3.3 鍵管理方針
[🔙 目次に戻る](#index)


* KSKはオフライン管理
* ZSKはDNSサーバ上で管理
* 鍵生成・更新作業は二名体制

[🔙 目次に戻る](#index)
